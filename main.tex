\section{Introduction}

\section{Preliminaries and Technical Overview}
\label{sec:prelim}
Let $n$ be the security parameter. Let $\mathcal{G}$ be a PPT algorithm that, on input $1^n$, outputs a (multiplicative) Abelian group $\group$ whose order is superpolynomial, and where uniformly sampling, multiplication and inverse can be done in polynomial time. (We do not require $\group$ to be cyclic, and do not assume $|\group|$ to be public.) We implicitly assume that $\group \gets \mathcal{G}(1^n)$ is run as part of the initialization process. Furthermore, we assume that all algorithms take $1^n$ and $\group$ as inputs, and do not explicitly write them.
\subsection{KEM Properties Used in the ``Hash Everything'' Version}
\paragraph{One-wayness and a variant.}
This property roughly says that given a public key and a ciphertext, an adversary cannot feasibly find the key encapsulated in the ciphertext.
\begin{definition}
A KEM scheme is \emph{one-way (OW)} if for any PPT adversary $\adv$, the probability that $\adv$ wins the following game is negligible:
\[
    \fcodebox{
    $(pk,sk) \gets \Gen$ \\
    $(c,K) \gets \Encap(pk)$ \\
    $K^* \gets \adv(pk,c)$ \\
    $\adv$ wins if $K^* = K$
    }
\]
\end{definition}
We also need a stronger variant in which the adversary is additionally given access to a \emph{plaintext-checking oracle}, which tells the adversary whether a ciphertext decapsulates to a certain key.
\begin{definition}
The definition of \emph{one-wayness under one plaintext-checking attack (OW-1PCA)} is the same as that of OW, except that the adversary $\adv$ is additionally given access to an oracle $\PC(sk,\star,\star)$, which on input $(c' \neq c, K')$ outputs 1 if $K' = \Decap(sk,c')$ and 0 otherwise. $\adv$ is allowed to query this oracle only once.
\end{definition}
This property is the same as what's called OW-PCA in \cite[Definition~3]{EPRINT:ArrBarJar25}, except that in our notion (1) the adversary can query its plaintext-checking oracle only once, and (2) the adversary is prohibited from querying the challenge ciphertext to its plaintext-checking oracle. The plaintext-checking oracle was first studied in \cite{PKC:AbdBenPoi15} in the context of public-key encryption schemes.

OW-1PCA is needed in the scenario where the OEKE adversary $\adv$ forwards the $P$-to-$P'$ message $(s,T)$ without modification, but then modifies the $P'$-to-$P$ message $(c,\tau')$ to some other $(c^*,\tau^*)$. Consider the time when $P'$ outputs its session key $SK'$: at this point $\adv$ has not done any attack yet, so in the ideal world the PAKE functionality will set $SK'$ at random. This means that $\adv$ must not be able to compute $K'$ (otherwise it can query $\Hkey(K')$ and distinguish $SK'$ from random); furthermore, forward secrecy requires that $\adv$ should not compute $K'$ even if it learns $\pw'$ later (which allows $\adv$ to decrypt $(s,T)$ and learn $pk'$) --- which is why we give the public key to the adversary in the security game.

Furthermore, later when $\adv$ modifies $(c,\tau')$ to $(c^* \neq c, \tau^* = \Htag(\dots,K^*))$ for $K^*$ of its choice, it can check if $K^* = \Decap(sk,c^*)$ by observing if $P$'s session key $SK = \Hkey(K^*)$. This corresponds to the (single) query to the plaintext-checking oracle. (This requires that $P$ and $P'$'s passwords match; otherwise $sk$ on the $P$ side does not correspond to $pk'$ on the $P'$ side.)

\paragraph{Anonymity and a variant.}
This property roughly says that given a public key and a ciphertext, an adversary cannot feasibly tell whether the ciphertext corresponds to this public key or some other (freshly sampled) public key.
\begin{definition}
A KEM scheme is \emph{anonymous (ANO)} if for any PPT adversary $\adv$, $\Pr[b^* = 1]$ in the following two games are negligibly close:\footnote{The challenge bit $b$ is not actually used in the definition, but it will be handy in the later security proofs.}
\[
    \fcodebox{
    \underline{challenge bit $b = 0$}: \\
    $(pk,sk) \gets \Gen$ \\
    $(c,\star) \gets \Encap(pk)$ \\
    $b^* \gets \adv(pk,c)$
    }
    \quad
    \fcodebox{
    \underline{challenge bit $b = 1$}: \\
    $(pk,sk) \gets \Gen$ \\
    $(pk',\star) \gets \Gen$ \\
    $(c,\star) \gets \Encap(pk')$ \\
    $b^* \gets \adv(pk,c)$
    }
    \]
\end{definition}
Similar to OW-1PCA, we also need the 1PCA variant of ANO.
\begin{definition}
The definition of \emph{anonymity under one plaintext-checking attack (ANO-1PCA)} is the same as that of ANO, except that the adversary $\adv$ is additionally given access to an oracle $\PC(sk,\star,\star)$, which on input $(c' \neq c, K')$ outputs 1 if $K' = \Decap(sk,c')$ and 0 otherwise. $\adv$ is allowed to query this oracle only once.
\end{definition}
This property is the same as what's called ANO-PCA in \cite[Definition~4]{EPRINT:ArrBarJar25}, except that in our notion (1) the adversary can query its plaintext-checking oracle only once, and (2) the adversary is not given $pk'$ (whereas in their notion $pk'$ is generated and given to the adversary in both games).

ANO-1PCA is needed again in the scenario where the OEKE adversary $\adv$ forwards the $P$-to-$P'$ message $(s,T)$ without modification, but then modifies the $P'$-to-$P$ message $(c,\tau')$. Consider $c$: in the real world $c$ is the result of $\Encap(pk)$ (assuming the two parties' passwords match), whereas in the ideal world the simulator does not know $pk$ at this point (it learns no information about $\pw$, so it cannot decrypt $(s,T)$ and learn $pk$) and has to simulate $c$ as the result of $\Encap(pk')$ for a fresh $pk'$. $\adv$ should not be able to tell these two cases apart (even after it learns $\pw$ later and uses it to decrypt $(s,T)$ and learn $pk$), which is ANO. 1PCA is needed for the same reason as in OW-1PCA.

\paragraph{Anonymity with uniform public keys and random oracles.}
Finally, we define a highly tailored KEM property that is implied by the preceding more standard properties. This property is for facilitating the security proof of OEKE only; we do not expect it to be of independent interest.
\begin{definition}
\label{def:uni-ano-ro}
A KEM scheme is \emph{anonymous with $q$ uniform public keys and random oracles ($q$-UNI-ANO-RO)} if $\Pr[b^* = 1]$ in the following two games are negligibly close:
\[
    \fcodebox{
    \underline{challenge bit $b = 0$}: \\
    $pk_1, \dots, pk_q \gets \pkspace$ \\
    $i \gets \adv(pk_1, \dots, pk_q)$ \\
    $(c,K) \gets \Encap(pk_i)$ \\
    $h_1 := H_1(K); h_2 := H_2(K)$ \\
    $b^* \gets \adv(c,h_1,h_2)$
    }
    \quad
    \fcodebox{
    \underline{challenge bit $b = 1$}: \\
    $pk',pk_1, \dots, pk_q \gets \pkspace$ \\
    $i \gets \adv(pk_1, \dots, pk_q)$ \\
    $(c,\star) \gets \Encap(pk')$ \\
    $h_1 \gets \{0,1\}^n; h_2 \gets \{0,1\}^{2n}$ \\
    $b^* \gets \adv(c,h_1,h_2)$
    }
    \]
Here, $H_1$ and $H_2$ are ROs with range $\{0,1\}^n$ and $\{0,1\}^{2n}$, respectively.
\end{definition}

This property emerges when the OEKE adversary $\adv$ sends $(s^*,T^*)$ to $P'$ which contains a wrong password guess. In the real world, $(s^*,T^*)$ decrypt to a uniformly random KEM public key, which is then used to compute $c,SK,\tau$ (the left game). In simulation, we want to remove the decryption process and outright use a freshly sampled public key to compute $c,SK,\tau$ (the right game). ($h_1,h_2$ in \Cref{def:uni-ano-ro} correspond to $SK,\tau$ in the protocol, respectively.) What complicates things is that $\adv$ can decrypt a large number of $(s^*,T^*)$ pairs \emph{under both correct and incorrect passwords}, resulting in a large number of $pk^*$ values, \emph{before} sending $(s^*,T^*)$ to $P'$ (or even before initiating $P'$'s session); at this time the reduction does not know which $(s^*,T^*)$ will be used by $\adv$, or which password is correct, yet it has to program the public key (the ``right'' one that will be used in $P'$'s algorithm) into the decryption process. As such, we need to give the reduction polynomially many public keys.

\subsection{Additional KEM Properties Used in Other Versions}
\begin{definition}
A KEM scheme is \emph{weakly non-malleable (WNM)} if for any PPT adversary $\adv$, the probability that $\adv$ wins the following game is negligible:
\[
    \fcodebox{
    $(pk,sk) \gets \Gen$ \\
    $(c,K) \gets \Encap(pk)$ \\
    $c^* \gets \adv(pk,c)$ \\
    abort if $c^* = c$ \\
    $\adv$ wins if $\Decap(sk,c^*) = K$
    }
\]
\end{definition}

\begin{definition}
A KEM scheme is \emph{unpredictable (UNPRE)} if for any PPT adversary $\adv$, the probability that $\adv$ wins the following game is negligible:
\[
    \fcodebox{
    $(\star,sk) \gets \Gen$ \\
    $(c^*,K^*) \gets \adv()$ \\
    $\adv$ wins if $\Decap(sk,c^*) = K^*$
    }
\]
\end{definition}

\begin{definition}
A KEM scheme is \emph{output key binding (KBIND)} if for any PPT adversary $\adv$, the probability that $\adv$ wins the following game is negligible:
\[
    \fcodebox{
    $(pk,sk) \gets \Gen$ \\
    $pk' \gets \pkspace$ \\
    $(c,K') \gets \Encap(pk')$ \\
    $c^* \gets \adv(pk,pk',c,K')$ \\
    $\adv$ wins if $\Decap(sk,c^*) = K'$
    }
\]
\end{definition}

\begin{definition}
A KEM scheme is \emph{robust (ROB)} if for any PPT adversary $\adv$, the probability that $\adv$ wins the following game is negligible:
\[
    \fcodebox{
    $(pk,sk) \gets \Gen$ \\
    $(pk',sk') \gets \Gen$ \\
    $c^* \gets \adv(pk,pk')$ \\
    $\adv$ wins if $\Decap(sk,c^*) = \Decap(sk',c^*)$
    }
\]
\end{definition}

\subsection{UC Description of the OEKE Protocol}
\label{sec:protocol-uc}
We give a formal, UC-compatible description of the OEKE protocol in \Cref{fig:protocol-uc}. Note that UC PAKE considers the case that the two parties' passwords are different; therefore, we need to use different notations $\pw$ and $\pw'$ for the password of $P$ and $P'$, respectively.\footnote{\cite[Fig.4]{EPRINT:ArrBarJar25} uses $\pw$ for both parties' passwords, which is not the ``right'' protocol description in the UC setting.} We only describe the ``hash everything'' version of the protocol; in other variants one or more of $\pw$, $pk$, $(s,T)$ and $c$ can be removed from the inputs to $\Htag$.

\begin{figure}[H]
\begin{framed}
\begin{small}
\underline{Party $P$, step 1}

\medskip
On input $(\NewSession, sid, P', \pw, \text{``initializer''})$, generate $(pk,sk) \gets \Gen$, sample $r \gets \{0,1\}^{3n}$, compute
\begin{align*}
R &:= H_1(\pw,r), &T &:= pk \cdot R, \\
t &:= H_2(\pw,T), &s &:= r \oplus t,
\end{align*}
and send $(sid,s,T)$ to $P'$. Store $\langle\pw,pk,sk,s,T\rangle$ associated with $sid$.

\medskip
\underline{Party $P'$}

\medskip
On input $(\NewSession, sid, P, \pw', \text{``respondent''})$ and $(sid,s,T)$ from $P$, compute
\begin{align*}
t' &:= H_2(\pw',T), &r' &:= s \oplus t', \\
R' &:= H_1(\pw',r'), &pk' &:= T/R',
\end{align*}
$(c,K') \gets \Encap(pk')$, $SK' := \Hkey(K')$, and $\tau' := \Htag(\pw',pk',s,T,c,K')$, send $(sid,c,\tau')$ to $P$, and output $(sid,SK')$.

\medskip
\underline{Party $P$, step 2}

\medskip
On $(sid,c,\tau')$ from $P'$, retrieve $\langle\pw,pk,sk,s,T\rangle$ associated with $sid$. Then compute $K := \Decap(sk,c)$, $\tau := \Htag(\pw,pk,s,T,c,K)$, and
\[
SK :=
\begin{cases}
\Hkey(K) & \text{if }\tau = \tau' \\
\bot & \text{if }\tau \neq \tau'
\end{cases}
\]
and output $(sid,SK)$. \quad \textcolor[gray]{0.5}{// No need to store $\pw$, $pk$ or $(s,T)$ in step 1 if not included in $\Htag$ (but always need to store $sk$)}
\end{small}
\end{framed}
\caption{UC description of the OEKE protocol (the ``hash everything'' version)}
\label{fig:protocol-uc}
\end{figure}

\paragraph{Correctness.}
Suppose $\pw = \pw'$ and there is no adversary. Then $P'$ computes
\begin{align*}
t' &= H_2(\pw',T) = t, &r' &= s \oplus t' = r, \\
R' &= H_1(\pw',r') = R, &pk' &= T/R' = pk,
\end{align*}
so $(c,K')$ is the result of $\Encap(pk')$. Then on the $P$ side, $P$ computes $K := \Decap(sk,c)$, which is equal to $K'$ except with probability $\varepsilon_\text{correctness}$ (the correctness error of the KEM). If $K = K'$ then $\tau = \tau'$ so $P$'s check passes, and $P$ will output $SK = SK'$. Therefore, the OEKE protocol has correctness error $\varepsilon_\text{correctness}$.

\section{Security Analysis of the ``Hash Everything'' Version}
\label{sec:basic}

The main theorem in this section is:
\begin{theorem}
\label{thm:basic}
Suppose the UNI-PK, OW-1PCA and ANO-1PCA properties hold for the KEM scheme. Then the OEKE protocol as described in \Cref{fig:protocol-uc} UC-realizes $\Fpake$, in the random oracle model for $H_1$, $H_2$, $\Hkey$ and $\Htag$.
\end{theorem}
The rest of this section is dedicated to the proof of \Cref{thm:basic}. In \Cref{sec:simulator} we describe the simulator, and in \Cref{sec:games} we argue why the simulator generates a view indistinguishable from the real-world view.

As standard in UC proofs, w.l.o.g. we only construct a simulator $\sim$ for the ``dummy'' adversary $\adv$ that merely forwards all messages between the environment $\env$ and protocol parties \cite[Claim~11]{FOCS:Canetti01}. We omit $sid$ in protocol messages and honest parties' outputs (e.g., we say ``$P$ outputs $SK$'' rather than ``$P$ outputs $(sid,SK)$'').

\subsection{The Simulator}
\label{sec:simulator}
\subsubsection{Simulation Strategy}
\label{sec:sim-intuition}
A large portion of the simulator is routine; below we explain the idea behind the non-trivial parts.
\paragraph{Simulating honest $P$-to-$P'$ message.}
When $P$'s session begins, $\sim$ must simulate its message $(s,T)$ to $P'$. This can be easily done: just sample $(s,T)$ at random. However, every time $\adv$ queries $H_2(\pw^*,T)$ (let the result be $t^*$), $\sim$ needs to generate a KEM key pair $(pk^*,sk^*)$ for later use (see the ``Extracting password guess from malicious $P'$'' paragraph below) and make sure that $(pk^*,sk^*)$ is consistent with the RO queries. This can be done as follows: $\sim$ computes
\[
r^* := s \oplus t^*, \quad R^* := T/pk^*,
\]
and programs $H_1(\pw^*,r^*) := R^*$. This also allows $\sim$ to store the association between $\pw^*$ and $(pk^*,sk^*)$. (Note that $(pk^*,sk^*)$ needs to be freshly generated every time $\adv$ queries $H_2(\star,T)$; otherwise $R^*$ never changes and $\adv$ can easily find collisions in $H_1$.)
\paragraph{Extracting password guess from malicious $P$.}
When the adversary $\adv$ sends a message $(s^*,T^*)$ to $P'$, if it is different from the honest message $(s,T)$, the simulator $\sim$ must extract a password guess (if any) and send a corresponding $\TestPwd$ command to $\Fpake$. There are two types of attack, which need to be addressed separately:
\begin{enumerate}
  \item \emph{``Normal'' online attack:} $\adv$ executes $P$'s algorithm on a password guess $\pw^*$. That is, $\adv$ generates KEM public key $pk^*$, samples randomness $r^*$, computes
      \begin{align*}
      R^* &:= H_1(\pw^*,r^*), &T^* &:= pk^* \cdot R^*, \\
      t^* &:= H_2(\pw^*,T^*), &s^* &:= r^* \oplus t^*,
      \end{align*}
      and sends $(s^*,T^*)$ to $P'$. \\
      It takes a while to realize how to extract $\pw^*$ from $(s^*,T^*)$. First, $\sim$ should scan all $H_2(\star,T^*)$ queries, which yields multiple $\pw^*$ values; $\sim$ must decide which one is the ``actual'' password guess. This can be done via the following. For every $H_2(\pw^*,T^*)$ query (let the result be $t^*$), $\sim$ computes the corresponding $r^* := s^* \oplus t^*$, and checks if (and when) $H_1(\pw^*,r^*)$ was queried. \emph{With overwhelming probability, there is at most one such $H_1$ query made before the $H_2(\pw^*,T^*)$ query}, from which $\sim$ can extract the ``actual'' password guess $\pw^*$.
  \item \emph{Related key attack:} This attack works only after $\adv$ receive an honest $(s,T)$ pair from $P$. $\adv$ skips generating $pk^*$ or sampling $r^*$; instead, $\adv$ picks any $\Delta \in \group$, computes $T^* := T \cdot \Delta$ and
      \begin{align*}
      t^* &:= H_2(\pw^*,T), &(t^*)' &:= H_2(\pw^*,T^*), \\
      \delta &:= t^* \oplus (t')^*, &s^* &:= s \oplus \delta,
      \end{align*}
      and sends $(s^*,T^*)$ to $P'$. \\
      Let us first explain what $\adv$ is trying to achieve here. The same randomness $r$ yields two valid messages, $(s,T)$ and $(s^*,T^*)$, such that
      \[
      s \oplus s^* = H_2(\pw,T) \oplus H_2(\pw,T^*),
      \]
      which allows $\adv$ to pick $T^*$ and ``reverse engineer'' the corresponding $s^*$ if it guesses $\pw$ correctly. $(s^*,T^*)$ implicitly encrypts $pk^* = T^*/H_1(\pw,r)$, which $\adv$ can compute as $pk \cdot (T^*/T)$. \\
      To extract $\pw^*$ from $(s^*,T^*)$, $\sim$ should scan all $H_2(\star,T)$ and $H_2(\star,T^*)$ queries (which yield multiple candidate $\pw^*$ values) and check which ones satisfy
      \[
      H_2(\pw^*,T) \oplus H_2(\pw^*,T^*) = s \oplus s^*.
      \]
      With overwhelming probability, at most one $\pw^*$ will satisfy this equation.
\end{enumerate}

\paragraph{Remark: On the term ``anchor query''.}
The term ``anchor query'' was coined in \cite[Section~3.3]{CCS:McQRosRoy20}, where it refers to the special $H_1(\pw^*,r^*)$ query made before the $H_2(\pw^*,T^*)$ query in case (1) above; case (2) is completely missing in \cite{CCS:McQRosRoy20}. This error is fixed in \cite{EC:JanRoyXu25}, which uses ``anchor query'' to refer to something else: the $H_2(\pw^*,T^*)$ query in either case (1) or (2) (see \cite[Section~4.3.1]{EC:JanRoyXu25} and \cite[Figure~11,~step~4]{EC:JanRoyXu25}). This term is not used in \cite{EPRINT:ArrBarJar25}.

\paragraph{Extracting password guess from malicious $P'$.}
When the adversary $\adv$ sends a message $(c^*,\tau^*)$ to $P$, if $\adv$ is not passive (i.e., it is not the case that $(s^*,T^*) = (s,T)$ \emph{and} $(c^*,\tau^*) = (c,\tau)$), the simulator $\sim$ must extract a password guess (if any) and send a corresponding $\TestPwd$ command to $\Fpake$.

$\sim$ should find the $\Htag$ query resulting in $\tau^*$, which contains $(\pw^*,pk^*,K^*)$. However, $\pw^*$ constitutes a password guess only if it is consistent with $pk^*$ and $K^*$, i.e., $K^* = \Decap(sk^*,c^*)$ for $sk^*$ corresponding to $pk^*$. $\sim$ can check this condition because it has stored the correspondence between $\pw^*$ and $(pk^*,sk^*)$ (see the ``Simulating honest $P$-to-$P'$ message'' paragraph above), and can use $sk^*$ to check if the equation holds. (This extraction strategy does not work anymore if \emph{either} $\pw^*$ \emph{or} $pk^*$ is not included in $\Htag$. In later sections we will explain how to simulate other versions of the protocol.)

\subsubsection{Formal Description of the Simulator}
\label{sec:sim}

Below we use $\func$ as a shorthand for $\Fpake$. Please see \Cref{fig:oeke-sim1,fig:oeke-sim2} for the simulator $\sim$.

As notational conventions, we write ``$\sim$ sends message $m$ from $P$ to $P'$'' as a shorthand for ``$\sim$ sends $m$ to $\env$ that pretends to be a real-world protocol message from $P$ to $P'$, intercepted by $\adv$ which in turn forwards it to $\env$''. Similarly, we write ``$\sim$ receives message $m$ from $\adv$ to $P$'' for ``$\env$ instructs $\adv$ to send $m$ to $P$, and in the ideal world it is $\sim$ who receives $m$ from $\env$''.

\begin{figure}[H]
\begin{framed}
\begin{small}
\underline{$P$-to-$P'$ message}

\medskip
On $(\NewSession, sid, P, P', \text{``initiator''})$ from $\func$, sample $s \gets \{0,1\}^{3n}, T \gets \group$ and send $(s,T)$ from $P$ to $P'$.

\medskip
\underline{$P'$'s session key and $P'$-to-$P$ message}

\medskip
On $(\NewSession, sid, P', P, \text{``respondent''})$ from $\func$ and $(s^*,T^*)$ from $\adv$ to $P'$, simulate $P'$'s output and the $P'$-to-$P$ message as follows:
\begin{enumerate}
  \item If $(s^*,T^*) = (s,T)$, send $(\NewKey, sid, P', 0^n)$ to $\func$; generate $(pk',\star) \gets \Gen$, compute $(c,\star) \gets \Encap(pk')$, sample $\tau' \gets \{0,1\}^{2n}$, and send $(c',\tau')$ from $P'$ to $P$.
  \item Otherwise find $\adv$'s $H_2((\pw')^*,T^*)$ query (let the result be $t^*$) that satisfies \emph{either} of the following:
      \begin{itemize}
        \item \emph{Type (1):} $\adv$ queried $H_1((\pw')^*,r^*)$ \emph{before} the $H_2((\pw')^*,T^*)$ query, where $r^* = s^* \oplus t^*$.
        \item \emph{Type (2):} $\adv$ also queried $H_2((\pw')^*,T)$, and
            \[
            s \oplus s^* = H_2((\pw')^*,T) \oplus t^*.
            \]
      \end{itemize}
      \begin{enumerate}
        \item If there is more than one such $(\pw')^*$, abort.
        \item If there is exactly one such $(\pw')^*$, send $(\TestPwd, sid, P', (\pw')^*)$ to $\func$; if there is no such $(\pw')^*$, send $(\TestPwd, sid, P', \bot)$ to $\func$.
            \begin{enumerate}
              \item If $\func$ replies with ``wrong guess'', send $(\NewKey, sid, P', 0^n)$ to $\func$; {\color{red}sample $pk' \gets \group$}, compute $(c,\star) \gets \Encap(pk')$, sample $\tau' \gets \{0,1\}^{2n}$, and send $(c,\tau')$ from $P'$ to $P$.
              \item If $\func$ replies with ``correct guess'', and $(\pw')^*$ is defined in type (1) above, then $r^*$ is already defined. Compute
                  \[
                  R' := H_1((\pw')^*,r^*), \quad pk' := T^*/R'.
                  \]
                  If $\func$ replies with ``correct guess'', and $(\pw')^*$ is defined in type (2) above, then $\adv$ has queried $H_2((\pw')^*,T)$, and a record $\langle(\pw')^*,(pk')^*,\star\rangle$ must have been stored at that time (see the ``RO queries'' paragraph below). Compute
                  \[
                  pk' := (pk')^* \cdot (T^*/T).

\subsection{Game-Hop Argument}
\label{sec:games}
We now prove that for any PPT environment $\env$, the simulator $\sim$ in \Cref{sec:sim} generates an ideal-world view indistinguishable from $\env$'s real-world view. The sequence of games starts from the real world (Game 0) and ends in the ideal world (Game 17).

A game-by-game comparison with the proof in \cite[Section~5]{EPRINT:ArrBarJar25} is shown in \emph{{\color{red}red}}, with the parts that critically rely on certain items being included as inputs to $\Htag$ \emph{{\color{red}\ul{underlined}}}.

\medskip\noindent\emph{Game 0:} This is the real world. Recall that the passwords of $P$ and $P'$ are $\pw$ and $\pw'$, respectively; the flow of events is:
\begin{itemize}
  \item $P$ sends $(s,T)$ to $P'$;
  \item It is intercepted by the man-in-the-middle adversary $\adv$, who sends $(s^*,T^*)$ (which may or may not be equal to $(s,T)$) to $P'$;
  \item $P'$ outputs session key $SK'$ and sends $(c,\tau')$ to $P$;
  \item It is again intercepted by $\adv$, who sends $(c^*,\tau^*)$ to $P$;
  \item $P$ computes $\tau$. If $\tau = \tau^*$ then $P$ outputs an $n$-bit string $SK$, otherwise $P$ outputs $\bot$.
\end{itemize}
(Note that there are three $\tau$ values: $\tau'$ computed and sent by $P'$, $\tau^*$ sent by $\adv$, and $\tau$ computed by $P$ used in $P$'s check.) A graphic illustration is shown in \Cref{fig:real-world}.

\medskip\noindent\emph{Game 1:} In the case where $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c$, do the following:
\begin{itemize}
  \item If $\tau^* = \tau'$ (i.e., $\adv$ is passive), set $SK := SK'$.
  \item If $\tau^* \neq \tau'$, set $SK := \bot$.
\end{itemize}
By correctness of the protocol (see \Cref{sec:protocol-uc}), in Game 0 $\tau = \tau'$ except with probability $\varepsilon_\text{correctness}$. This means that except with probability $\varepsilon_\text{correctness}$, $P$'s check $\tau^* \stackrel{?}{=} \tau$ in Game 0 is equivalent to $P$'s check $\tau^* \stackrel{?}{=} \tau'$ in Game 1. If the check passes then both games set $SK := \Hkey(K)$, otherwise both games set $SK := \bot$. We have
\[
\Dist_\env^{0,1} = \varepsilon_\text{correctness}.
\]

\emph{{\color{red}This game corresponds to games $\mathsf{G}_3$--$\mathsf{G}_5$ in \cite{EPRINT:ArrBarJar25}. We believe having a single game that handles both the $\tau^* = \tau'$ case and the $\tau^* \neq \tau'$ case is much cleaner.}}

\paragraph{$P$ and $P'$'s passwords match, and $\adv$ forwards the $P$-to-$P'$ message.}
\mbox{}

\medskip\noindent\emph{Game 2:} In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, find $\adv$'s $\Htag(\pw,pk,s,T,c^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. If there is a unique such query, do the following:
\begin{itemize}
  \item If $K^* = K$, set $SK := \Hkey(K)$.
  \item If $K^* \neq K$, set $SK := \bot$.
\end{itemize}

\begin{claim}
\label{clm:htag}
$P'$ does not query $\Htag(\pw,pk,s,T,c^*,K)$.
\end{claim}

$P'$ makes a single $\Htag$ query, on $(\pw',pk',s^*,T^*,c,K')$. Since $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, of course $P'$'s query is not $\Htag(\pw,pk,s,T,c^*,K)$.\footnote{The proof of this claim is trivial, but we explicitly present it as a claim since its counterparts in some variants are much harder to prove.}

\medskip
Going back to the game hop, Game 1 and Game 2 are identical unless one of the following happens:
\begin{enumerate}
  \item $\adv$ never makes an $\Htag(\pw,pk,s,T,c^*,\star)$ query resulting in $\tau^*$, yet $P$'s check $\tau^* \stackrel{?}{=} \Htag(\pw,pk,s,T,c^*,K)$ passes.
  \item $\adv$ makes more than one $\Htag(\pw,pk,s,T,c^*,\star)$ query resulting in $\tau^*$.
  \item $\adv$ makes an $\Htag(\pw,pk,s,T,c^*,K^*)$ query (where $K^* \neq K$) resulting in $\tau^*$, and $P$'s $\Htag(\pw,\allowbreak pk, s,T,c^*,K)$ query also results in $\tau^*$.
\end{enumerate}

In the first event, by \Cref{clm:htag} $P'$ does not query $\Htag(\pw,pk,s,T,c^*,K)$. $\adv$ itself also does not query $\Htag(\pw,pk,s,T,c^*,K)$ (otherwise the result is $\tau^*$). So $\Htag(\pw,pk,s,T,c^*,K)$ is independent of $\adv$'s view, and the probability that $\adv$ sends $\tau^* = \Htag(\pw,pk,s,T,c^*,K)$ is $1/2^{2n}$. The second and third events are collision in $\Htag$ from either $\adv$'s or $P$'s queries, whose (combined) probability is upper-bounded by $\qtag(\qtag+1)/2^{2n+1}$. Overall,
\[
\Dist_\env^{1,2} \leq \frac{\qtag(\qtag+1)+2}{2^{2n+1}}.
\]

\emph{{\color{red}This game corresponds to games $\mathsf{G}_6$ and $\mathsf{G}_7$ in \cite{EPRINT:ArrBarJar25}, except that the $\Htag$ collision cases (events (2) and (3) above) were ruled out in game $\mathsf{G}_2$ in \cite{EPRINT:ArrBarJar25} (as aborting condition [A1]). \\
\indent\ul{The proof of \mbox{\Cref{clm:htag}}
% has to do this \mbox trick since \ul directly on \Cref does not compile
critically relies on $\pw,s,T,c$ being included in $\Htag$ inputs.}}}

\medskip\noindent\emph{Game 3:} In the case where $\pw = \pw' \land (s^*,T^*) = (s,T)$, sample $SK',\tau' \gets \{0,1\}^n$. ($c$ is still computed as before.)

Let $\Query_1$ be the event that $\adv$ queries \emph{either} $\Hkey(K')$ \emph{or} $\Htag(\pw,pk,s,T,c,\allowbreak K')$. Clearly Game 2 and Game 3 are identical unless $\Query_1$ happens. We upper-bound $\Pr[\Query_1]$ via a reduction $\red_\text{OW-1PCA1}$ to the OW-1PCA property of KEM:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{OW\text{-}1PCA1}$}:} \quad \textcolor[gray]{0.5}{// \fbox{Boxed text} indicates the single $\PC$ oracle query; same for reductions below}

\medskip\noindent
Sample $i \gets [\qkey+\qtag]$ as a guess that $\adv$'s $i$-th $\Hkey$ or $\Htag$ query triggers $\Query_1$. \\
On input $(pk,c)$,
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, run $P$'s algorithm to compute $(s,T)$, except that $\red_\text{ANO-1PCA}$ uses its input $pk$ instead of $(pk,\star) \gets \Gen$. (Note that computing $(s,T)$ does not require knowing $sk$.) Send $(s,T)$ to $\adv$.
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $\neg(\pw = \pw' \land (s^*,T^*) = (s,T))$, abort. Otherwise sample $SK' \gets \{0,1\}^n, \tau' \gets \{0,1\}^{2n}$, output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$. ($c$ is part of $\red_\text{OW-1PCA1}$'s inputs.)
  \item On $(c^*,\tau^*)$ from $\adv$,
      \begin{enumerate}
        \item If $c^* = c$, do what's described in Game 1.
        \item If $c^* \neq c$, find $\adv$'s $\Htag(\pw,pk,s,T,c^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, output $SK := \bot$ to $\env$. (This matches Game 2.)
        \item \fbox{Query $\PC(sk,c^*,K^*)$}. \quad // \textcolor[gray]{0.5}{This is a valid query since we do this step only if $c^* \neq c$} \\
            If the result is 1, output $SK := \Hkey(K^*)$ to $\env$. \\
            If the result is 0, output $SK := \bot$ to $\env$.
      \end{enumerate}
  \item On $\adv$'s $i$-th $\Hkey$ or $\Htag$ query, if it is in the form of $\Hkey((K')^*)$ or $\Htag(\pw,pk,s,T,c,(K')^*)$ for some $(K')^*$, output $(K')^*$; otherwise abort.
\end{enumerate}

Assuming $\Query_1$ happens, the only difference between Game 2/3 and $\red_\text{OW-1PCA1}$'s simulation (until $\Query_1$ happens) is that $\red_\text{OW-1PCA1}$ uses its input $pk$ on the $P$ side, and then uses its input $c \gets \Encap(pk)$ on the $P'$ side. Since we assume $\pw = \pw' \land (s^*,T^*) = (s,T)$, in Game 2/3 $P'$'s algorithm will recover $pk' = pk$, so there $c$ is also computed as $\Encap(pk)$ and thus $\red_\text{OW-1PCA1}$ simulates Game 2/3 perfectly. Furthermore, if $\red_\text{OW-1PCA1}$'s guess is correct then it wins the OW-1PCA game. We have
\[
\Dist_\env^{2,3} \leq \Pr[\Query_1] \leq (\qkey+\qtag)\Adv_{\red_\text{OW-1PCA1}}^\text{OW-1PCA}.
\]

\emph{{\color{red}This game corresponds to game $\mathsf{G}_8$ in \cite{EPRINT:ArrBarJar25}. However, there are two differences:
\begin{enumerate}
  \item \cite{EPRINT:ArrBarJar25} reduces to OW-PCA (where the KEM adversary can query its plaintext-checking oracle polynomially many times), rather than OW-1PCA. In this case, step 4 of the reduction above can query $\PC(sk,c,(K')^*)$ $\qkey+\qtag$ times to check which of $\adv$'s queries (if any) triggers $\Query_1$, hence avoiding the $\qkey+\qtag$ loss in its advantage.
  \item The above also means that the plaintext-checking oracle in their OW-PCA game has to allow for $\PC(sk,c,\star)$ queries, whereas our reduction to OW-1PCA never makes a query in this form and thus the plaintext-checking oracle in our OW-1PCA game can disallow such queries.
\end{enumerate}
}}

\medskip\noindent\emph{Game 4:} Again in the case where $\pw = \pw' \land (s^*,T^*) = (s,T)$, generate $P'$'s KEM public key $(pk',\star) \gets \Gen$ and compute $(c,\star) \gets \Encap(pk')$. (Note that $K'$ is not used anywhere since Game 3 --- in particular, $P'$ does not use $K'$ to compute $SK'$ or $\tau'$ anymore --- so we do not need to explicitly mention it.)

The difference between Game 3 and Game 4 is that in Game 3 both $P$ and $P'$ use the same $pk$, whereas in Game 4 $P$ uses $pk$ and $P'$ uses an independent $pk'$. We construct a reduction $\red_\text{ANO-1PCA}$ to the ANO-1PCA property of KEM. This reduction, in fact, is identical to the reduction $\red_\text{OW-1PCA1}$ to the OW-1PCA property in Game 3 except the last step; for completeness, we still present the full reduction and highlight the difference in {\color{blue}blue}:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{ANO\text{-}1PCA}$}:}

\medskip\noindent
On input $(pk,c)$,
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, run $P$'s algorithm to compute $(s,T)$, except that $\red_\text{ANO-1PCA}$ uses its input $pk$ instead of $(pk,\star) \gets \Gen$. Send $(s,T)$ to $\adv$.
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $\neg(\pw = \pw' \land (s^*,T^*) = (s,T))$, output 1 and halt. Otherwise sample $SK' \gets \{0,1\}^n, \tau' \gets \{0,1\}^{2n}$, output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$. ($c$ is part of $\red_\text{ANO-PCA1}$'s inputs.)
  \item On $(c^*,\tau^*)$ from $\adv$,
      \begin{enumerate}
        \item If $c^* = c$, do what's described in Game 1.
        \item If $c^* \neq c$, find $\adv$'s $\Htag(\pw,pk,s,T,c^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, output $SK := \bot$ to $\env$.
        \item \fbox{Query $\PC(sk,c^*,K^*)$}. \quad // \textcolor[gray]{0.5}{This is a valid query since we do this step only if $c^* \neq c$} \\
            If the result is 1, output $SK := \Hkey(K^*)$ to $\env$. \\
            If the result is 0, output $SK := \bot$ to $\env$.
      \end{enumerate}
  \item {\color{blue}When $\env$ outputs bit $b^*$, copy $\env$'s output.}
\end{enumerate}

$\red_\text{ANO-1PCA}$ uses its input $pk$ on the $P$ side and $c$ on the $P'$ side. If $\red_\text{ANO-1PCA}$'s challenge bit $b = 0$ then $c \gets \Encap(pk)$, i.e., both $P$ and $P'$ use the same $pk$, so $\red_\text{ANO-1PCA}$ simulates Game 3; if $b = 1$ then $c \gets \Encap(pk')$, i.e., $P'$ uses an independently generated $pk'$, so $\red_\text{ANO-1PCA}$ simulates Game 4. We have
\[
\Dist_\env^{3,4} \leq \Adv_{\red_\text{ANO-1PCA}}^\text{ANO-1PCA}.
\]

\emph{{\color{red}This game corresponds to game $\mathsf{G}_9$ in \cite{EPRINT:ArrBarJar25}. However, there are two differences:
\begin{enumerate}
  \item \cite{EPRINT:ArrBarJar25} reduces to ANO-PCA (where the KEM adversary can query its plaintext-checking oracle polynomially many times), rather than ANO-1PCA. Unlike in Game 3, here reducing to this stronger assumption does not provide any advantage.
  \item The assumption \cite{EPRINT:ArrBarJar25} reduces to is even further stronger in that their KEM adversary's input is $(pk,pk',c)$, whereas we do not give $pk'$ to the KEM adversary. Note that the reduction does not need $pk'$ at all (it only needs to send $c$ which is an encapsulation of either $pk$ or $pk'$), so the assumption in \cite{EPRINT:ArrBarJar25} is unnecessarily strong.
\end{enumerate}
}}

\paragraph{$P$ and $P'$'s passwords do not match, and $\adv$ forwards the $P$-to-$P'$ message.}
\mbox{}

\medskip\noindent\emph{Game 5:} After $P$ sends $(s,T)$ aimed at $P'$, when $\adv$ makes a fresh query $H_2(\pw^*,T)$ for $\pw^* \neq \pw$ (let the result be $t^*$), generate $(pk^*,sk^*) \gets \Gen$, and compute
\[
r^* := s \oplus t^*, \quad R^* := T/pk^*.
\]
If $H_1(\pw^*,r^*)$ has already been defined and it is not $R^*$, abort. Otherwise program $H_1(\pw^*,r^*) := R^*$, and store $\langle\pw^*,pk^*,sk^*\rangle$.

Furthermore, in the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T)$, when $P'$ queries $H_2(\pw',T)$, use the method described above to generate $pk',r',R'$ and program $H_1$. (Note that the case where $\pw = \pw' \land (s^*,T^*) = (s,T)$ has already been addressed in Game 3 and Game 4, and $P'$ does not need to make an $H_2$ query there.)\footnote{A slightly confusing case is when $s^* \neq s \land T^* = T$, causing $P'$ to query $t' := H_2(\pw',T)$; and $\adv$ also queries $H_2(\pw',T)$. In this case $P'$'s $r' = s^* \oplus t'$ and $\adv$'s $r^* = s \oplus t'$ are different, causing $P'$'s values $R',pk'$ and $\adv$'s values $R^*,pk^*$ to be independent of each other. Therefore, the game should run $P'$'s algorithm as usual when $P'$ queries $H_2(\pw',T)$ (which generates $R',pk'$), and use the new method described in Game 5 when $\adv$ makes the same query (which generates $R^*,pk^*$).}

In the analysis below, we combine $\adv$'s $H_2(\pw^*,T)$ queries and $P'$'s (possible) $H_2(\pw',T)$ query, and call all of them ``$H_2(\pw^*,T)$''. We first upper-bound the aborting probability. For every fresh $H_2(\pw^*,T)$ query, an independent $t^* \gets \{0,1\}^{3n}$ value is sampled, which results in an independent $r^* \gets \{0,1\}^{3n}$ (regardless of the distribution of $s$). As such, there are up to $q_2+1$ independent and uniformly random $r^*$ values. The probability that one of $\adv$'s $q_1$ $H_1(\pw^*,r^*)$ query happens \emph{before} the $H_2(\pw^*,T)$ query (when $r^*$ is generated) is upper-bounded by $q_1(q_2+1)/2^{3n}$.\footnote{$P$ also makes an $H_1$ query, but it is on $(\pw,r)$ which is guaranteed to be different from $(\pw^*,r^*)$ (in the case where $\adv$ queries $H_2(\pw^*,T)$) and $(\pw',r')$ (in the case where $P'$ queries $H_2(\pw',T)$).}

Now suppose the aborting condition does not happen. The difference between Game 4 and Game 5 is that Game 4 samples $H_1(\pw^*,r^*)$ uniformly at random, whereas Game 5 defines it as $T/pk^*$. Note that
\begin{itemize}
  \item For $H_2(\pw^*,T)$ queries where $\pw^* \neq \pw'$, $sk^*$ (or even $pk^*$) is not used anywhere in the game;
  \item For the $H_2(\pw',T)$ query, we cannot say the above anymore, since $pk'$ is used while computing $c$ and $K'$. However, since $\pw \neq \pw'$, we know that $pk = T/H_1(\pw,r)$ is independent of $pk' = T/H_1(\pw',r')$, so $sk'$ is still not used anywhere else in the game.
\end{itemize}
So, the distinguishing advantage between Game 4 and Game 5 can be reduced to the UNI-PK property of KEM. We only sketch the reduction $\red_\text{UNI-PK}$ here: $\red_\text{UNI-PK}$, on input $pk^*$ (which is either an honestly generated public key or a uniformly random value), samples $i \gets [q_2+1]$, and answers the (either $\adv$'s or $P'$'s) first $i-1$ $H_2$ queries as in Game 4 and the last $q_2-i+1$ $H_2$ queries as in Game 5; for the $i$-th $H_2$ query, $\red_\text{UNI-PK}$ computes
\[
r^* := s \oplus t^*, \quad R^* := T/pk^*
\]
and programs $H_1(\pw^*,r^*) := R^*$. If this $H_2$ query is on $(\pw',T)$, $\red_\text{UNI-PK}$ also uses its input $pk^*$ to compute $c$ and $K'$. (Note that the $\langle\pw^*,pk^*,sk^*\rangle$ record is not actually used in Game 5, so $\red_\text{UNI-PK}$ does not need to store it.) $\red_\text{UNI-PK}$ simulates other parts of Game 4/5 honestly. Finally, $\red_\text{UNI-PK}$ copies $\env$'s output bit.

A standard hybrid argument shows that (assuming the aborting condition does not happen) $\env$'s distinguishing advantage between Game 4 and Game 5 is upper-bounded by $(q_2+1)\Adv_{\red_\text{UNI-PK}}^\text{UNI-PK}$.\footnote{Note that here we cannot make $q_2+1$ separate reductions to the UNI-PK property, and must reduce to UNI-PK ``in one blow''. Furthermore, the calculation of the reduction's advantage is more complicated than the normal case where the reduction wins as long as its guess is correct. This is similar to, say, why composing a PRG polynomial times yields another PRG; see, e.g., \cite[Section~3.4.3]{book:BonSho23} for details.} Overall, we have
\[
\Dist_\env^{4,5} \leq (q_2+1)\Adv_{\red_\text{UNI-PK}}^\text{UNI-PK} + \frac{q_1(q_2+1)}{2^{3n}}.
\]

\emph{{\color{red}This game corresponds to game $\mathsf{G}_{10}$ in \cite{EPRINT:ArrBarJar25}. However, there are two differences:
\begin{enumerate}
   \item \cite{EPRINT:ArrBarJar25} runs the procedure above to generate $pk',r',R'$ and program $H_1$ as long as $P'$ makes an $H_2(\pw',T)$ query --- that is, including the case where $s^* \neq s \land T^* = T$. (If $T^* \neq T$ then of course $P'$ does not make an $H_2(\pw',T)$ query and there is no change from the real world.) The reason for making this change is to prepare for Game 6 and Game 7 below, which simplifies $P'$'s algorithm when $\pw \neq \pw' \land (s^*,T^*) = (s,T)$ (i.e., if we did not make the change in Game 5 first, the reductions in Game 6 and Game 7 would not go through); therefore, it is unnecessary (and potentially confusing) to make this change when $s^* \neq s \land T^* = T$.
   \item The aborting case was ruled out in game $\mathsf{G}_2$ in \cite{EPRINT:ArrBarJar25} (as aborting condition [A2b1]).
\end{enumerate}
}}

\medskip\noindent\emph{Game 6:} In the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T)$, sample $SK',\tau' \gets \{0,1\}^n$. ($c$ is still computed as before.)

Let $\Query_2$ be the event that $\adv$ queries \emph{either} $\Hkey(K')$ \emph{or} $\Htag(\pw',pk',s,T,\allowbreak c,K')$. Clearly Game 5 and Game 6 are identical unless $\Query_2$ happens. We upper-bound $\Pr[\Query_2]$ via a reduction $\red_\text{OW1}$ to the OW property of KEM. This is somewhat similar to the reduction $\red_\text{OW-1PCA}$ to the OW-1PCA property in Game 3, and we highlight the differences in {\color{blue}blue}:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{OW1}$}:}

\medskip\noindent
Sample $i \gets [\qkey+\qtag]$ as a guess that $\adv$'s $i$-th $\Hkey$ or $\Htag$ query triggers $\Query_2$, {\color{blue}and $j \gets [q_2+1]$ as a guess that the $j$-th $H_2$ query (by either $\adv$ or $P'$) is $H_2(\pw',T)$}. \\
On input $(pk',c)$,
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, run $P$'s algorithm to compute $(s,T)$. Send $(s,T)$ to $\adv$.
  \item {\color{blue}Whenever there is an $H_2(\star,T)$ query (by either $\adv$ or $\red_\mathrm{OW1}$ itself on behalf of $P'$ --- see step 3 below), do what's described in Game 5. However, if this is the $j$-th query, use $\red_\mathrm{OW1}$'s input $pk'$ instead of generating $pk^*$ freshly.}
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $\neg({\color{blue}\pw \neq \pw'} \land (s^*,T^*) = (s,T))$, abort. Otherwise
      \begin{enumerate}
        {\color{blue}\item If $\adv$ has not queried $H_2(\pw',T)$, make this query on behalf of $P'$.}
        \item Sample $SK',\tau' \gets \{0,1\}^n$, output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$. ($c$ is part of $\red_\text{OW1}$'s inputs.)
      \end{enumerate}
  \item On $(c^*,\tau^*)$ from $\adv$,
      \begin{enumerate}
        \item Find $\adv$'s $\Htag(\pw,pk,c^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, output $SK := \bot$ to $\env$. (This matches Game 2.)
        \item {\color{blue}Compute $K := \Decap(sk,c)$. ($sk$ is the KEM secret key generated in step 1.)} \quad \textcolor[gray]{0.5}{// No need to query a plaintext-checking oracle since $pk$ on the $P$ side and $pk'$ on the $P'$ side are independent of each other, so the reduction can sample $sk$ on its own} \\
            If {\color{blue}$K = K^*$}, output $SK := \Hkey(K)$ to $\env$. \\
            If {\color{blue}$K \neq K^*$}, output $SK := \bot$ to $\env$.
      \end{enumerate}
  \item On $\adv$'s $i$-th $\Hkey$ or $\Htag$ query, if it is in the form of $\Hkey((K')^*)$ or $\Htag(\pw',pk',s,T,c,(K')^*)$ for some $(K')^*$, output $(K')^*$; otherwise abort.
\end{enumerate}

Assuming $\Query_2$ happens and $\red_\text{OW1}$'s guess of index $j$ is correct, the only difference between Game 5/6 and $\red_\text{OW1}$'s simulation (until $\Query_2$ happens) is that $\red_\text{OW1}$ uses its input $pk'$ to program $H_1(\pw',r')$, and then uses its input $(c,\star) \gets \Encap(pk')$ as the $P'$-to-$P$ message. Since $c$ is also computed as $\Encap(pk')$ in Game 5/6, $\red_\text{OW1}$ simulates Game 5/6 perfectly. Furthermore, if $\red_\text{OW1}$'s guess of index $i$ is correct then it wins the OW game. We have
\[
\Dist_\env^{5,6} \leq \Pr[\Query_2] \leq (q_2+1)(\qkey+\qtag)\Adv_{\red_\text{OW1}}^\text{OW}.
\]

\begin{remark}
It is very subtle why the reduction needs to guess the index $j$ and lose a factor of $q_2+1$ (and we applaud \cite{EPRINT:ArrBarJar25} for getting it right). The reason is as follows. Consider the following environment $\env$:
\begin{enumerate}
  \item Initiate $P$'s session on password $\pw$ and receive $(s,T)$.
  \item Instruct $\adv$ to make a large number of $H_2(\pw^*,T)$ queries (let the result be $t^*$), one of which is $H_2(\pw',T)$ for a special value $\pw'$.
  \item For each of the $H_2$ queries in step 2,
      \[
      \text{compute }r^* := s \oplus t^*, \quad \text{query }H_1(\pw^*,r^*).
      \]
  \item Initiate $P'$'s session on password $\pw'$ and instruct $\adv$ to forward $(s,T)$ to $P'$.
\end{enumerate}
To simulate Game 5/6 correctly, $\red_\text{OW1}$ must use its input $pk'$ to program $H_1(\pw',r') := T/pk'$ in step 3. The problem is that \emph{$\red_\mathrm{OW1}$ does not know $\pw'$ until step 4, so it has no idea which $H_1(\pw',r')$ query in step 3 needs to be programmed using $pk'$}! The only way to get around is to make a guess over all $H_2$ queries, which incurs a $q_2+1$ loss. (If we require $\env$ to always initiate $P$ and $P'$'s sessions simultaneously, this loss can be avoided since in step 3 $\red_\mathrm{OW1}$ must have already learned $\pw'$. However, the UC PAKE functionality does not place this restriction on $\env$.)
\end{remark}

\medskip\emph{{\color{red}This game corresponds to game $\mathsf{G}_{11}$ in \cite{EPRINT:ArrBarJar25}, except that they reduce to OW-PCA and avoids the $\qkey+\qtag$ loss (cf. the comment after Game 3).}}

\medskip\noindent\emph{Game 7:} Again in the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T)$, recall Game 6 does the following on the $P'$ side: {\color{brown}Generate $(pk',\star) \gets \Gen$, query $t' := H_2(\pw',T)$, and compute
\[
r' := s \oplus t', \quad R' := T/pk'.
\]
($(s,T)$ are the honest $P$-to-$P'$ message.) If $H_1(\pw',r')$ has already been defined and it is not $R'$, abort. Otherwise program $H_1(\pw',r') := R'$}, \ul{compute $(c,\star) \gets \Encap(pk')$}, and sample $SK',\tau' \gets \{0,1\}^n$. Finally, output $SK'$ and send $(c,\tau')$ to $P$. (Note that $K'$ is not used anywhere since Game 6 --- in particular, $P'$ does not use $K'$ to compute $SK'$ or $\tau'$ anymore --- so we do not need to explicitly mention it.) In this game, we independently generate $(pk'',\star) \gets \Gen$ and compute $(c,\star) \gets \Encap(pk'')$ (the underlined expression).

The difference between Game 6 and Game 7 is that in Game 6 $c$ is generated by
\[
pk' = \frac{T}{H_1(\pw',s \oplus H_2(\pw',T))}
\]
which $\adv$ can compute, whereas in Game 7 $c$ is generated by an independent $pk''$. We construct a reduction $\red_\text{ANO}$ to the ANO property of KEM. This reduction, in fact, is identical to the reduction $\red_\text{OW1}$ to the OW property in Game 6 except the last step; for completeness, we still present the full reduction and highlight the difference in {\color{blue}blue}:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{ANO}$}:}

\medskip\noindent
Sample $j \gets [q_2+1]$ as a guess that the $j$-th $H_2$ query (by either $\adv$ or $P'$) is $H_2(\pw',T)$. \\
On input $(pk',c)$,
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, run $P$'s algorithm to compute $(s,T)$. Send $(s,T)$ to $\adv$.
  \item Whenever there is an $H_2(\star,T)$ query (by either $\adv$ or $\red_\mathrm{ANO1}$ itself on behalf of $P'$ --- see step 3 below), do what's described in Game 5. However, if this is the $j$-th query, use $\red_\mathrm{ANO}$'s input $pk'$ instead of generating $pk^*$ freshly.
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $\neg(\pw \neq \pw' \land (s^*,T^*) = (s,T))$, abort. Otherwise
      \begin{enumerate}
        \item If $\adv$ has not queried $H_2(\pw',T)$, make this query on behalf of $P'$.
        \item Sample $SK',\tau' \gets \{0,1\}^n$, output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$. ($c$ is part of $\red_\text{ANO}$'s inputs.)
      \end{enumerate}
  \item On $(c^*,\tau^*)$ from $\adv$,
      \begin{enumerate}
        \item Find $\adv$'s $\Htag(\pw,pk,s,T,c^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, output $SK := \bot$ to $\env$.
        \item Compute $K := \Decap(sk,c^*)$. ($sk$ is the KEM secret key generated in step 1.) \quad \textcolor[gray]{0.5}{// No need to query a plaintext-checking oracle since $pk$ on the $P$ side and $pk'$ on the $P'$ side are independent of each other, so the reduction can sample $sk$ on its own} \\
            If $K = K^*$, output $SK := \Hkey(K)$ to $\env$. \\
            If $K \neq K^*$, output $SK := \bot$ to $\env$.
      \end{enumerate}
  \item {\color{blue}When $\env$ outputs bit $b^*$, copy $\env$'s output.}
\end{enumerate}

Assume $\red_\text{ANO}$'s guess of index $j$ is correct. If $b = 0$ then $c \gets \Encap(pk')$, i.e., the computation of $R'$ and $c$ use the same $pk'$, so $\red_\text{ANO}$ simulates Game 6; if $b = 1$ then $c \gets \Encap(pk'')$, i.e., the computation of $c$ uses an independently generated $pk''$, so $\red_\text{ANO}$ simulates Game 7. We have
\[
\Dist_\env^{6,7} \leq (q_2+1)\Adv_{\red_\text{ANO}}^\text{ANO}.
\]

\medskip\noindent\emph{Game 8:} Again in the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T)$, we outright skip the computation of $t',r',R'$ and the programming of $H_1$ (the {\color{brown}brown} text in Game 7). (Since $pk'$ is not generated anymore, we can rename $pk''$ --- the KEM public key from which $c$ is computed --- to $pk'$.)

Note that the aborting condition, which was originally introduced in Game 5, is removed (for the specific $H_2(\pw',T)$ query by $P'$). By an analysis similar to that in Game 5, the aborting probability in Game 7 is upper-bounded by $q_1/2^{3n}$. If the aborting condition does not happen, the {\color{brown}brown} text is independent of the rest of the game, so removing it does not affect the distribution of $\env$'s output. We have
\[
\Dist_\env^{6,7} \leq \frac{q_1}{2^{3n}}.
\]

\emph{{\color{red}The two games above combined correspond to game $\mathsf{G}_{12}$ in \cite{EPRINT:ArrBarJar25}. \cite{EPRINT:ArrBarJar25} does not have the intermediate Game 7 in our proof, and outright removes the brown text from what's our Game 6. We believe it is clearer to include our Game 7, where $H_2$ is still queried and $H_1$ is still programmed: in this way it is easier to see why the reduction to ANO simulates the games correctly. Also, \cite{EPRINT:ArrBarJar25} reduces to ANO-PCA\footnote{\cite{EPRINT:ArrBarJar25} actually mentions ANO-CPA, which appears to be a typo.}, although it also says that the plaintext-checking oracle is not needed.}}

\paragraph{Computation of the $P$-to-$P'$ message.}
\mbox{}

\medskip\noindent\emph{Game 9:} Note that up to now we have not changed $P$'s algorithm before sending $(s,T)$ (in particular, Game 5 dealt with $H_2$ queries by $\adv$ and $P'$, but not $P$). In this game we modify it as follows: Sample $s \gets \{0,1\}^{3n}, T \gets \group$. If $\adv$ has queried $H_2(\star,T)$, abort. {\color{brown}Otherwise query $t := H_2(\pw,T)$, generate $(pk,sk) \gets \Gen$, and compute
\[
r := s \oplus t, \quad R := T/pk.
\]
If $H_1(\pw,r)$ has already been defined and it is not $R$, abort. Otherwise program $H_1(\pw,r) := R$.}

We first upper-bound the aborting probability. Apart from the one made by $P$, there are $q_2$ $H_2$ queries in total (note that $P'$ does not query $H_2$ anymore since Game 8). Thus, the probability that one of the query inputs happens to contain $T$ is upper-bounded by $q_2/|\group|$. By an analysis similar to that in Game 5 and Game 8, the probability of the second aborting event is upper-bounded by $q_1/2^{3n}$.

Now suppose neither of the aborting events happens. The following equations hold in both Game 8 and Game 9:
\[
r = s \oplus H_2(\pw,T), \quad T = R \cdot pk.
\]
Here, $pk$ is generated by $\Gen$, and $H_2(\pw,T)$ is fixed by an $H_2$ query once $T$ is fixed. The four remaining variables $r,s,R,T$ are constrained by two equations and have $4-2=2$ degrees of freedom: if we sample $r \gets \{0,1\}^{3n}, R \gets \group$ then we get Game 8, and if we sample $s \gets \{0,1\}^{3n}, T \gets \group$ then we get Game 9. Thus, Game 8 and Game 9 are identical. We have
\[
\Dist_\env^{8,9} \leq \frac{q_1}{2^{3n}} + \frac{q_2}{|\group|}.
\]

\emph{{\color{red}The game above corresponds to game $\mathsf{G}_{13}$ in \cite{EPRINT:ArrBarJar25}, with one difference: our game aborts if $\adv$ has queried $H_2(\pw^*,T)$ for any $\pw^*$ when $T$ is sampled, whereas the game in \cite{EPRINT:ArrBarJar25} aborts only if $\adv$ has queried $H_2(\pw,T)$. Note that the simulator does not know $\pw$, so it cannot tell if $\adv$ has queried $H_2(\pw,T)$ and instead has to abort as long as $\adv$ has queried $H_2(\star,T)$. (The simulator in \cite{EPRINT:ArrBarJar25} is correct, but its game hops do not match the ideal world regarding which $H_2$ queries will trigger an abortion.) Also, we believe our argument under this game is much more concise and cleaner (our presentation is, in spirit, similar to \cite[Lemma~4.5,~Hybrid~2]{EC:JanRoyXu25}).}}

\medskip\noindent\emph{Game 10:} When $P$'s session begins, halt after checking if $H_2(\star,T)$ has been defined (i.e., before the {\color{brown} brown} text in Game 9). Then when $P$ receives $(c^*,\tau^*)$, if $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c$, do what's described in Game 1. Otherwise,
\begin{itemize}
  \item If $H_2(\pw,T)$ has now been defined (via a query by $\adv$), there must be a corresponding record $\langle\pw,pk,sk\rangle$ (see Game 5).
  \item If $H_2(\pw,T)$ is still undefined, run the {\color{brown} brown} text in Game 9.
\end{itemize}
Either way, $pk,sk$ are defined and the game can proceed as usual.

The only difference between Game 9 and Game 10 lies in the fact that when we move the {\color{brown} brown} text around, the game might abort --- as part of the {\color{brown} brown} text --- at different times. Since the aborting event happens with probability at most $q_1/2^{3n}$, we have
\[
\Dist_\env^{9,10} \leq \frac{q_1}{2^{3n}}.
\]

\emph{{\color{red}The game above corresponds to game $\mathsf{G}_{14}$ in \cite{EPRINT:ArrBarJar25}. Note that we have a $q_1/2^{3n}$ loss in both Game 9 and Game 10; if we remove Game 9 and directly jump to Game 10 from Game 8, this loss does not need to be repeated (since Game 8 and Game 10 are identical unless one of the two aborting events happens). We (as well as \cite{EPRINT:ArrBarJar25}) choose to include Game 9 for clarity.}}

\begin{enumerate}
  \item \emph{$P$-to-$P'$ message:} Game 10 outright samples $(s,T)$ at random. [changed in Game 9]
  \item \emph{$P'$'s session key and $P'$-to-$P$ message:} On $(s^*,T^*)$ from $\adv$ to $P'$, if $(s^*,T^*) = (s,T)$, Game 10 outright samples $SK',\tau'$ at random. \paragraph{Password extraction from $P'$-to-$P$ message.}
\mbox{}

\medskip\noindent\emph{Game 11:} In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, if there is no $\langle\pw,\star,\star\rangle$ record, set $SK := \bot$ (i.e., replace the {\color{brown}brown} text in \Cref{fig:intermediate-game} with ``$SK := \bot$'').

Game 10 and Game 11 differ only if $\adv$ does not make an $H_2(\pw,T)$ query, but makes an $\Htag(\pw,pk,s,T,c^*,K^*)$ query resulting in $\tau^*$ (for $c^*,K^*,\tau^*$ of $\adv$'s choice). This in particular requires $\adv$ to know $pk$, which is freshly generated by $\Gen$. Therefore, the probability that (even a computationally unbounded) $\adv$ can predict $pk$ is upper-bounded by $H_\infty(pk : (pk,\star) \gets \Gen)$. We have
\[
\Dist_\env^{10,11} \leq H_\infty(pk : (pk,\star) \gets \Gen).
\]

\emph{{\color{red}The game above corresponds to game $\mathsf{G}_{15}$ in \cite{EPRINT:ArrBarJar25}. \cite{EPRINT:ArrBarJar25} does not include the condition $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, i.e., it appears that this game rewrites what our Game 2 does, which is incorrect. \\
\indent\ul{The argument in this game critically relies on $pk$ being included in $\Htag$ inputs.}
}}

\medskip\noindent\emph{Game 12:} In the case where $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c \land \tau^* \neq \tau'$, do what's in the $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$ case.

In Game 11 $SK$ is outright set to $\bot$, whereas here it is treated the same as the ``normal'' case. Let $\mathsf{Query}_3$ be the event that $\adv$ queries $\Htag(\pw,pk,s,T,c,K)$, where $pk$ is the KEM public key generated on $\adv$'s $H_2(\pw,T)$ query; Game 11 and Game 12 are identical unless $\mathsf{Query}_3$ happens. To trigger $\mathsf{Query}_3$, $\adv$ must know $K$, which suggests a reduction $\red_\text{OW2}$ to the OW property of KEM. This reduction is simple, so we only provide a sketch: $\red_\text{OW2}$, on inputs $(pk,c)$, invokes $\env$ and simulates $(s,T)$ and the $P'$ side honestly. (Note that $P'$ uses a freshly generated $pk'$ independent of the $P$ side.) When $\adv$ queries $H_2(\pw,T)$, $\red_\text{OW2}$ uses its input $pk$ in lieu of $(pk,\star) \gets \Gen$. Along the way, as long as $\red_\text{OW2}$ detects that $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c \land \tau^* \neq \tau'$ does not happen, it aborts. Finally, $\red_\text{OW2}$ finds the (unique) $\adv$'s $\Htag(\pw,pk,s,T,c,K^*)$ query resulting in $\tau^*$, and outputs $K^*$. We have
\[
\Dist_\env^{11,12} \leq \Pr[\Query_3] \leq \Adv_{\red_\text{OW2}}^\text{OW}.
\]

\emph{{\color{red}The game above is missing in \cite{EPRINT:ArrBarJar25}. Note that in the ideal world, the simulator cannot tell if $\pw = \pw'$, so when it sees $(s^*,T^*) = (s,T) \land c^* = c \land \tau^* \neq \tau'$, it has to always try to extract a password guess; hence, the game above has to be here. (In the $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c \land \tau^* \neq \tau'$ case, we first outright set $SK := \bot$ in Game 1 and then ``walk back'' in Game 12, since the change in Game 1 is necessary for the reductions in Game 3 and Game 4 to go through.)}}

\medskip\noindent\emph{Game 13:} In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land (c^*,\tau^*) = (c,\tau))$, find $\adv$'s $\Htag(\pw^*,pk^*,s,T,c^*,K^*)$ query resulting in $\tau^*$. (The difference is that Game 12 only goes over all of $\adv$'s queries in the form of $\Htag(\pw,pk,s,T,c^*,\star)$, whereas this game goes over all of $\adv$'s queries in the form of $\Htag(\star,\star,s,T,c^*,\star)$.) If there is none or more than one, set $SK := \bot$. Otherwise there is a unique such $\Htag(\pw^*,pk^*,s,T,c^*,K^*)$ query, and
\begin{itemize}
  \item If $\pw^* \neq \pw$, set $SK := \bot$.
  \item If $\pw^* = \pw$, there is no change from Game 12.
\end{itemize}

If $\adv$ makes no $\Htag(\star,\star,s,T,c^*,\star)$ query resulting in $\tau^*$, then in particular $\adv$ makes no $\Htag(\pw,pk,s,T,c^*,\star)$ query resulting in $\tau^*$, so both Game 12 and Game 13 set $SK := \bot$. The probability that $\adv$ makes two different $\Htag(\star,\star,s,T,c^*,\star)$ queries both resulting in $\tau^*$ is upper-bounded by $\qtag(\qtag-1)/2^{2n+1}$. Now suppose $\adv$ makes a unique $\Htag(\pw^*,pk^*,s,T,c^*,K^*)$ query resulting in $\tau^*$:
\begin{itemize}
  \item If $\pw^* \neq \pw$, then $\adv$ does not make a $\Htag(\pw,pk,s,T,c^*,\star)$ query resulting in $\tau^*$ (because there is only one $\Htag(\star,\star,s,T,c^*,\star)$ query resulting in $\tau^*$, and it is on $\pw^*$), so both Game 12 and Game 13 set $SK := \bot$.
  \item If $\pw^* = \pw$, then there is no difference between Game 12 and Game 13.
\end{itemize}

We conclude that
\[
\Dist_\env^{12,13} \leq \frac{\qtag(\qtag-1)}{2^{2n+1}}.
\]

The $\Htag$ collision case was ruled out in game $\mathsf{G}_2$ in \cite{EPRINT:ArrBarJar25} (as aborting condition [A1])\footnote{In our game hops, we ruled out $\Htag$ collisions for different $K^*$ values in Game 2, and then additionally rule out $\Htag$ collisions for different $(\pw^*,pk^*,K^*)$ pairs in Game 13. \medskip\noindent\emph{Game 14:} In the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T) \land (c^*,\tau^*) = (c,\tau')$, set $SK := \bot$.

In this case $\tau^* = \tau'$ is uniformly random and independent of the rest of the game, so the probability that $\adv$ makes an $\Htag$ query resulting in $\tau^*$ is at most $\qtag/2^{2n}$. If $\adv$ does not make such a query, then both Game 13 and Game 14 set $SK := \bot$. We have
\[
\Dist_\env^{13,14} \leq \frac{\qtag}{2^{2n}}.
\]

\paragraph{Password extraction from $P$-to-$P'$ message.}
\mbox{}

\medskip\noindent\emph{Game 15:} In the case where $(s^*,T^*) \neq (s,T)$, find $\adv$'s $H_2(\pw^*,T^*)$ query\footnote{\label{footnote:notations}The notations are slightly messed up, since the $\pw^*$ here has nothing to do with what's called $\pw^*$ in Game 13. Essentially, the $\pw^*$ here is $\adv$'s guess of $P'$'s password (which is called $(\pw')^*$ in the simulator), whereas the $\pw^*$ in Game 13 is $\adv$'s guess of $P$'s password. The good news is that the $\pw^*$ in Game 13 is never reused once Game 13 is done, so there is no potential confusion.} (let the result be $t^*$) that satisfies \emph{either} of the following:
\begin{enumerate}
  \item $\adv$ queried $H_1(\pw^*,r^*)$ \emph{before} the $H_2(\pw^*,T^*)$ query, where $r^* = s^* \oplus t^*$.
  \item $\adv$ also queried $H_2(\pw^*,T)$, and
      \[
      s \oplus s^* = H_2(\pw^*,T) \oplus t^*.\footnote{Note that in this case $T^* \neq T$ since otherwise $T^* = T \Rightarrow H_2(\pw^*,T) = t^* \Rightarrow s^* = s \oplus H_2(\pw^*,T) \oplus t^* = s$, so $(s^*,T^*) = (s,T)$ which violates the condition of this game.}
      \]
\end{enumerate}
(This $H_2(\pw^*,T^*)$ query is what's called the ``anchor query'' in \cite{EC:JanRoyXu25}.) If there is more than one such query, abort.

The aborting condition is met only in the following three cases:
\begin{itemize}
  \item There is a collision in type (1) above, i.e., there are $r^*,t^*,(r^*)',(t^*)'$ such that
      \[
      r^* \oplus t^* = (r^*)' \oplus (t^*)'
      \]
      (so $\adv$ can set $s^*$ to be this value). Note that all four values in the equation are determined by two (different) $H_1$ queries and one $H_2$ query, and for each set of these queries, there is a $1/2^{3n}$ chance that the equation will hold. Therefore, the probability of this case is upper-bounded by $q_1(q_1-1)q_2/2^{3n}$. %\footnote{A sloppier argument would say that there are $q_1$ possible $r^*$ values and $q_2$ possible $t^*$ values, so there are $q_1q_2$ possible $r^* \oplus t^*$ values and the probability of a collision is upper-bounded by $q_1q_2(q_1q_2-1)/2^{3n+1}$. This yields a looser bound since those $r^* \oplus t^*$ values are not all independent of each other. The same goes for the two cases below.}
  \item There is a collision in type (2) above, i.e., there are $\pw^*,t^*,(\pw^*)',(t^*)'$ such that
      \[
      H_2(\pw^*,T) \oplus t^* = H_2((\pw^*)',T) \oplus (t^*)'
      \]
      (so $\adv$ can set $s^*$ to be $s \oplus H_2(\pw^*,T) \oplus t^*$). Note that all four values in the equation are determined by three (different) $H_2$ queries $H_2(\pw^*,T)$, $H_2((\pw^*)',T)$ and $H_2(\pw^*,T^*)$; by an argument similar to above, the probability of this case is upper-bounded by $q_2(q_2-1)(q_2-2)/2^{3n}$.
  \item There is a collision between type (1) and type (2), i.e., there are $r^*,t^*,(\pw^*)',\allowbreak(t^*)'$ such that
      \[
      r^* \oplus t^* = s \oplus H_2((\pw^*)',T) \oplus (t^*)'
      \]
      (so $\adv$ can set $s^*$ to be this value). Note that $s$ is already fixed, and all four other values in the equation are determined by two (different) $H_2$ queries and one $H_1$ query; by an argument similar to above, the probability of this case is upper-bounded by $q_1q_2(q_2-1)/2^{3n}$.
\end{itemize}

Summing up, we have
\[
\Dist_\env^{14,15} \leq \frac{q_2[q_1(q_1+q_2-2)+(q_2-1)(q_2-2)]}{2^{3n}}.
\]

\emph{{\color{red}This game corresponds to part of game $\mathsf{G}_2$ in \cite{EPRINT:ArrBarJar25} (as aborting conditions [A2a] and [A2b2]). The game in \cite{EPRINT:ArrBarJar25} aborts more often than necessary, though: every time $\adv$ makes an $H_2(\star,\star)$ query, the game performs the checks and aborts accordingly, while it is only necessary to go over all of $\adv$'s $H_2(\star,T^*)$ queries for the specific $T^*$ as part of $\adv$'s message to $P'$. \\
\indent Performing the checks on $\adv$'s $H_2$ query, instead of when $\adv$ sends the $(s^*,T^*)$ message, also adds more bookkeeping and complicates the proof. In \cite{EPRINT:ArrBarJar25}, after performing the checks, the game (if it does not abort) computes the corresponding KEM public key and stores it together with $(\pw^*,T^*)$ (the record is called $\mathsf{ForwS}$); later when $\adv$ sends $(s^*,T^*)$, the game first fetches the record to recover $\pw^*$, and if $\pw^* = \pw'$, the game then recovers the KEM public key and uses it to complete the remaining part of $P'$'s algorithm. This creates a conceptual discrepancy with the ideal world, and a game hop (game $\mathsf{G}_{16}$ in \cite{EPRINT:ArrBarJar25}) is needed. By performing the checks when $\adv$ sends the $(s^*,T^*)$, we avoid all these and skip their game $\mathsf{G}_{16}$.
}}

\paragraph{Correct password guess for $P'$.}
\mbox{}

\medskip\noindent\emph{Game 16:} Again in the case where $(s^*,T^*) \neq (s,T)$, if $\pw^*$ is defined in type (2) in Game 15 and $\pw^* = \pw'$, retrieve record $\langle\pw^*,pk^*,\star\rangle$ (there must be one since such a record is created when $\adv$ queries $H_2(\pw^*,T)$, as per Game 5) and set $pk' := pk^* \cdot (T^*/T)$.\footnote{What's called $pk^*$ here is $(pk')^*$ in the simulator, as there is another $pk^*$ on the $P$ side as defined in Game 13 (cf. \Cref{footnote:notations}).}

In Game 15, we have (in $P'$'s algorithm)
\[
pk' = \frac{T^*}{R'} = \frac{T^*}{H_1(\pw',r')} = \frac{T^*}{H_1(\pw',s^* \oplus t')} = \frac{T^*}{H_1(\pw',s^* \oplus H_2(\pw',T^*))},
\]
but also (in the creation of the record $\langle\pw^*,pk^*,\star\rangle$)
\[
pk^* = \frac{T}{R^*} = \frac{T}{H_1(\pw^*,r^*)} = \frac{T}{H_1(\pw^*,s \oplus t^*)} = \frac{T}{H_1(\pw^*,s \oplus H_2(\pw^*,T))}.
\]
The condition of this game specifies that $\pw^* = \pw'$, and type (2) in Game 15 specifies that $s \oplus H_2(\pw^*,T) = s \oplus H_2(\pw^*,T^*)$. This means that the denominators in the expressions of $pk'$ and $pk^*$ are equal, and thus $pk' = pk^* \cdot (T^*/T)$ --- exactly as in Game 15. We have
\[
\Dist_\env^{15,16} = 0.
\]

\emph{{\color{red}This game corresponds to game $\mathsf{G}_{17}$ in \cite{EPRINT:ArrBarJar25}.}}

\paragraph{Incorrect password guess for $P'$.}
\mbox{}

\medskip\noindent\emph{Game 17:} Again in the case where $(s^*,T^*) \neq (s,T)$, if $\pw^* \neq \pw'$ or no $\pw^*$ is defined, sample $pk' \gets \group$, compute $(c,\star) \gets \Encap(pk')$, and sample $SK',\tau' \gets \{0,1\}^n$.

We construct a reduction $\red_\text{UNI-ANO-RO}$ to the $(q_1+1)(q_2+1)$-UNI-ANO-RO property of KEM:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{UNI\text{-}ANO\text{-}RO}$}:}

\medskip\noindent
On inputs $(pk_{1,1},\dots,pk_{q_1+1,q_2+1})$,
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, sample $s \gets \{0,1\}^{3n}, T \gets \group$. Send $(s,T)$ to $\adv$.
  \item Whenever there is an $H_2(\star,T)$ query by $\adv$, do what's described in Game 5. Furthermore, for the $j$-th query to $H_2$ (by $\adv$ or $\red_\text{UNI-ANO-RO}$ itself on behalf of $P'$ --- see step 3 below), say it is $H_2(\pw^*,T^*) =: t^*$, and the $i$-th $H_1$ query in the form of $H_1(\pw^*,r^*)$, compute
      \[
      s^* := r^* \oplus t^*, \quad R^* := T^*/pk_{i,j}.
      \]
      \begin{enumerate}
        \item If $H_1(\pw^*,r^*)$ has been defined when the $H_2(\pw^*,T^*)$ query happens (either because it was queried \emph{before} the $H_2(\pw^*,T^*) =: t^*$ query or because it has been programmed by $\red_\text{UNI-ANO-RO}$ while answering $\adv$'s $H_2(\pw^*,T)$ query), store $\langle\mathsf{PasswordGuess},\pw^*,s^*,T^*\rangle$. \quad // \textcolor[gray]{0.5}{Anchor query, i.e., $\pw^*$ is the password guess corresponding to $(s^*,T^*)$} \\
            At any time, if there is more than one $\langle\star,s^*,T^*\rangle$ record with the same $s^*,T^*$, abort.
        \item Otherwise program $H_1(\pw^*,r^*) := R^*$.
      \end{enumerate}
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $(s^*,T^*) = (s,T)$, output 1. If there is a record $\langle\mathsf{PasswordGuess},\pw',s^*,T^*\rangle$, also output 1. Otherwise \quad // \textcolor[gray]{0.5}{Incorrect or no password guess (the case that matters in this game hop)}
      \begin{enumerate}
        \item If $\adv$ has not queried $H_2(\pw',T^*)$ or $H_1(\pw',r')$ (where $r' = s^* \oplus H_2(\pw',T^*)$), make these queries on behalf of $P'$.
        \item Say $H_2(\pw',T^*)$ is the $j^*$-th query to $H_2$ and $H_1(\pw',r')$ is the $i^*$-th query to $H_1$ in the form of $H_1(\pw',\star)$. Output $(i^*,j^*)$ to the challenger and receive $(c,SK',\tau')$.
        \item Output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$.
      \end{enumerate}
  \item On $(c^*,\tau^*)$ from $\adv$,
      \begin{enumerate}
        \item Retrieve record $\langle\pw,pk,sk\rangle$. If there is no such record, output $SK := \bot$ to $\env$.
        \item Find $\adv$'s $\Htag(\pw,pk,s,T,c^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, output $SK := \bot$ to $\env$.
        \item Compute $K := \Decap(sk,c^*)$. ($sk$ is the KEM secret key generated in step 1.) \quad \textcolor[gray]{0.5}{// No need to query a plaintext-checking oracle since $pk$ on the $P$ side and $pk'$ on the $P'$ side are independent of each other, so the reduction can sample $sk$ on its own} \\
            If $K = K^*$, output $SK := \Hkey(K)$ to $\env$. \\
            If $K \neq K^*$, output $SK := \bot$ to $\env$.
      \end{enumerate}
  \item When $\env$ copies a bit $b^*$, copy $\env$'s output.
\end{enumerate}

We first argue that the $\pw^*$ as in record $\langle\mathsf{PasswordGuess},\pw^*,s^*,T^*\rangle$ is exactly the same as the $\pw^*$ extracted from $(s^*,T^*)$ in Game 15:
\begin{itemize}
  \item If $\adv$ queried $H_1(\pw^*,r^*)$ before the $H_2(\pw^*,T^*)$ query, this is exactly type (1) in Game 15.
  \item If $\adv$ queried $H_2(\pw^*,T)$, which triggered the programming of $H_1(\pw^*,r^*)$ before the $H_2(\pw^*,T^*)$ query, note that the $H_1$ instance that is programmed on is $(\pw^*, s \oplus H_2(\pw^*,T))$. Thus,
      \[
      s \oplus H_2(\pw^*,T) = r^* = s^* \oplus H_2(\pw^*,T^*),
      \]
      which is exactly type (2) in Game 15.
\end{itemize}
Therefore, $\red_\text{UNI-ANO-RO}$ accurately decides whether the condition ``$\pw^* \neq \pw'$ or no $\pw^*$ is defined'' is satisfied or not.

Next, we argue that $\red_\text{UNI-ANO-RO}$ simulates Game 16/17 correctly:
\begin{itemize}
  \item Suppose $\red_\text{UNI-ANO-RO}$ is in the ``real'' game where $(c,K') \gets \Encap(pk_{i^*,j^*})$ and $SK' = \Hkey(K'), \tau' = \Htag(\dots,K')$. For the ``special'' $H_2(\pw',T^*)$ query and $H_1(\pw',r')$ query, in both Game 16 and $\red_\text{UNI-ANO-RO}$'s simulation,
      \[
      s^* = r' \oplus H_2(\pw',T^*), \quad t^* = pk' \cdot H_1(\pw',r');
      \]
      if we sample $H_1(\pw',r') \gets \group$ then we get Game 16, whereas if we sample $pk' \gets \group$ (as $pk_{i^*,j^*}$) then we get $\red_\text{UNI-ANO-RO}$'s simulation. Later, $pk'$ is used to compute $c,K'$ in both Game 16 and $\red_\text{UNI-ANO-RO}$'s simulation. \\
      Regarding other $H_2(\pw^*,T^*)$ and $H_1(\pw^*,r^*)$ queries, the $pk_{i,j}$ that $\red_\text{UNI-ANO-RO}$ uses while programming $H_1(\pw^*,r^*)$ is uniformly random and independent of the rest of the game, so $H_1(\pw^*,r^*)$ is programmed to a uniformly random value --- again the same as Game 16.
  \item Suppose $\red_\text{UNI-ANO-RO}$ is in the ``random'' game where $(c,K') \gets \Encap(pk')$ and $SK' = \Hkey(K'), \tau' = \Htag(\dots,K')$ for a freshly sampled $pk'$. \emph{All} $pk_{i,j}$ values that $\red_\text{UNI-ANO-RO}$ uses while programming $H_1$ are uniformly random and independent of the rest of the game, so $H_1$ is always programmed to a uniformly random value --- the same as Game 17. Later, a freshly sampled $pk'$ is used to compute $c,K'$ in both Game 17 and $\red_\text{UNI-ANO-RO}$'s simulation.
\end{itemize}
We have
\[
\Dist_\env^{16,17} \leq \Adv_{\red_\text{UNI-ANO-RO}}^{(q_1+1)(q_2+1)\text{-UNI-ANO-RO}}.
\]

\emph{{\color{red}This game corresponds to games $\mathsf{G}_{18}$ and $\mathsf{G}_{19}$ in \cite{EPRINT:ArrBarJar25}. \cite{EPRINT:ArrBarJar25} has two games: $\mathsf{G}_{18}$ that changes $SK'$ and $\tau'$ (via a reduction to the OW-PCA property of KEM), and $\mathsf{G}_{19}$ that changes $c$ (via a reduction to the ANO-PCA property of KEM). While we believe their argument is fundamentally correct, it could be significantly improved:
\begin{enumerate}
  \item \cite{EPRINT:ArrBarJar25} reduces to OW-PCA and ANO-PCA\footnote{\cite{EPRINT:ArrBarJar25} actually mentions ANO-CPA, which appears to be a typo.}, which is unnecessary: OW and ANO suffice.
  \item Their simulation strategy is slightly different from ours: they use $pk'$ generated by $\Gen$, rather than sampled uniformly at random. Note that in this ``incorrect password guess'' case, in the real world $pk' = T^*/H_1(\pw',r')$ is already uniformly random, so this is fundamentally a KEM property about encapsulation of uniformly random (rather than honestly generated) public keys and we believe it is clearer to present it in this way.\footnote{By contrast, in the $(s^*,T^*) = (s,T)$ case $pk'$ has to be generated honestly, which is our Games 3, 4, 6 and 7: If $\pw = \pw' \land (s^*,T^*) = (s,T)$ then $sk$ corresponding to $pk = pk'$ is used on the $P$ side (in decapsulation), so $pk'$ cannot be changed to uniformly random; and the $\pw \neq \pw' \land (s^*,T^*) = (s,T)$ case cannot be separated from the $\pw = \pw' \land (s^*,T^*) = (s,T)$ case (since the simulator never knows if $\pw = \pw'$ or not), so $pk'$ cannot be changed to uniformly random in this case either.}
  \item Their honest generation of $pk'$ also means that their reductions cannot simulate the games perfectly, since they program $H_1(\pw',r')$ using $pk'$ which is not uniformly random --- while it ``should'' be uniformly random. As such, they have do a reduction to UNI-PK within the OW-PCA/ANO-PCA reduction.
  \item What makes things even worse is that none of their (or our) reductions knows which of the $H_1(\pw',r')$ queries should be programmed using $pk'$, since $\adv$ can make many decryption efforts, resulting in many $H_1$ values to be programmed, \emph{before} sending $(s^*,T^*)$ to $P'$ (cf. discussion after \Cref{def:uni-ano-ro}). This means that their UNI-PK reduction cannot be moved ``out of'' the OW-PCA/ANO-PCA reduction by defining a separate game that uses honestly generated $pk'$ to program $H_1(\pw',r')$, as that would require the \emph{game} to make a guess! What's going on here is essentially a hybrid argument (in the sense of \cite[Section~3.4.3]{book:BonSho23}), which is not explained clearly in \cite{EPRINT:ArrBarJar25}.
  \item Finally, the reductions in \cite{EPRINT:ArrBarJar25} appear to never check for the ``related key attack'', i.e., from their proof it is unclear what will go wrong if we remove what's our Game 16 (or their game $\mathsf{G}_{17}$).
\end{enumerate}
}}

\paragraph{Comparison with the ideal world.}

\section{Security Analyses of Other Versions (I): Including $(s,T)$ in the hash}
We assume the reader is familiar with the security proof in \Cref{sec:basic}, as the simulator and many games there will be frequently referred to or reused.

\subsection{What Will Go Wrong If You Don't Hash Everything?}
One might fear that every new variant of OEKE --- with some inputs removed from $\Htag$ --- needs a new security analysis from scratch, causing this paper to be hundreds of pages long. Fortunately, most of the simulator and the game hops from the proof in \Cref{sec:basic} can be reused. This, in fact, should be expected: only the definition of the $P'$-to-$P$ authenticator $\tau'$ changes in these variants, so the only parts of the proof that are affected are $P$'s behavior --- in particular, how to extract the password guess and when to output $\bot$ --- on the $P'$-to-$P$ message.

Concretely, Games 2, 11 and 13 in \Cref{sec:games} need stronger KEM security properties, or at least a new analysis, to go through. We explained in \emph{{\color{red}\ul{underlined}}} comments in each of them why the current arguments critically require $\pw$, $pk$, $(s,T)$ and/or $c$ as part of the $\Htag$ inputs, which we summarize in \Cref{tab:game-summary}:

\hline
 11 & \makecell{reject if $\adv$ does not make \\ an $H_2(\pw,T)$ query} & $pk$ \\
\hline
 13 & \makecell{extract $\pw^*$ from $\tau^*$ \\ and check against $K^*$} & $\pw$ \\

In this section, we analyze seven variants: those with $(s,T)$ included in $\Htag$, but at least one of $\pw,pk,c$ removed. In \Cref{app:no-st} we analyze the variants where $(s,T)$ are removed from $\Htag$.

\subsection{The ``Hash $\pw$, $pk$ and $(s,T)$'' Version}
\label{sec:hash-pw-pk-st}
We start from the version where $\tau$ is defined as $\Htag(\pw,pk,s,T,K)$, i.e., $c$ is removed from the hash. \Cref{tab:game-summary} suggests that only Game 2 needs to change, and the simulator and all other games can be reused in the proof.

Looking more closely, only \Cref{clm:htag} in Game 2 needs a new proof: the argument after this claim also does not change if we remove $c$ from $\Htag$. Below we state and prove the new claim.
\begin{claim}
\label{clm:htag2}
Suppose the WNM property holds for the KEM scheme. In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, the probability that $P'$ queries $\Htag(\pw,pk,s,T,K)$ is negligible.
\end{claim}
$P'$ makes a single $\Htag$ query, on $(\pw',pk',s,T,K')$. If $\pw \neq \pw' \lor (s^*,T^*) \neq (s,T)$, then of course $P'$'s query is not $\Htag(\pw,pk,s,T,K)$. Now assume $\pw = \pw' \land (s^*,T^*) = (s,T)$. This implies that $pk = pk'$ by the definitions of $pk$ and $pk'$, and $c^* \neq c$ by the condition of this game. We now upper-bound $\Pr[K = K']$. Recall that $K$ is defined as $\Decap(sk,c^*)$, so $\Pr[K = K'] = \Pr[\Decap(sk,c^*) = K']$. We upper-bound this probability via a reduction $\red_\text{WNM}$ to the WNM property of KEM:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{WNM}$}:}

\medskip\noindent
On input $(pk,c)$,
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, run $P$'s algorithm to compute $(s,T)$, except that $\red_\mathrm{WNM}$ uses its input $pk$ instead of $(pk,\star) \gets \Gen$. Send $(s,T)$ to $\adv$.
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $\neg(\pw = \pw' \land (s^*,T^*) = (s,T))$, abort. Otherwise generate $(c,K') \gets \Encap(pk)$, and compute $SK := \Hkey(K')$ and $\tau' := \Htag(\pw,pk,s,T,c,K')$. Output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$.
  \item On $(c^*,\tau^*)$ from $\adv$, if $c^* = c$, abort. Otherwise output $c^*$.
\end{enumerate}

The definition of $\red_\text{WNM}$ ensures that $\Decap(sk,c) = K'$. If $\adv$ sends $c^*$ such that $ \Decap(sk,c^*) = K'$, then $\red_\text{WNM}$ wins the WNM game. Thus,
\[
\Pr[\Decap(sk,c^*) = K'] \leq \Adv_{\red_\text{WNM}}^\text{WNM}.
\]
If $\Decap(sk,c) \neq K'$, i.e., $K \neq K'$, then of course $P'$'s query is not $\Htag(\pw,pk,s,\allowbreak T,K)$. This yields the claim.

\subsection{The ``Hash $\pw$ and $(s,T)$'' Version}
\label{sec:hash-pw-st}
We now turn to the variant where $pk$ is further removed from $\Htag$ inputs, i.e., $\tau$ is defined as $\Htag(\pw,s,T,K)$. \Cref{tab:game-summary} suggests that the argument under Game 11 in \Cref{sec:games} need to (essentially) change. (Of course, since $c$ is also removed from $\Htag$, the change in \Cref{sec:hash-pw-pk-st} also needs to be made.) Below we repeat Game 11 and make the new argument.

\medskip\noindent\emph{Game 11:} In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, if there is no $\langle\pw,\star,\star\rangle$ record, set $SK := \bot$ (i.e., replace the {\color{brown}brown} text in \Cref{fig:intermediate-game} with ``$SK := \bot$'').

Game 10 and Game 11 differ only if $\adv$ does not make an $H_2(\pw,T)$ query, but makes an $\Htag(\pw,s,T,K^*)$ query resulting in $\tau^*$, where $K^* = \Decap(sk,c^*)$ for a freshly generated $sk$. We upper-bound this probability via a reduction $\red_\text{UNPRE}$ to the UNPRE property of KEM:

\medskip\noindent\textbf{\underline{Reduction $\red_\mathrm{UNPRE}$}:}

\medskip\noindent
\begin{enumerate}
  \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, sample $s \gets \{0,1\}^{3n}, T \gets \group$. Send $(s,T)$ to $\adv$.
  \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, run $P'$'s algorithm in Game 10 to compute $c$, $SK'$ and $\tau'$. Output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$.
  \item On $(c^*,\tau^*)$ from $\adv$, if $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c$, abort. Otherwise find $\adv$'s $\Htag(\pw,s,T,K^*)$ query resulting in $\tau^*$. If there is none or more than one, abort. Otherwise output $(c^*,k^*)$.
  \item On $\adv$'s $H_2(\pw^*,T)$ query, if $\pw^* = \pw$, abort. Otherwise do what Game 10 specifies.
\end{enumerate}

It is straightforward to see that if the ``bad event'' above happens, then $\red_\mathrm{UNPRE}$ wins its UNPRE game. We have
\[
\Dist_\env^{10,11} \leq \Adv_{\red_\text{UNPRE}}^\text{UNPRE}.
\]

Overall, we conclude that the ``hash $\pw$ and $(s,T)$'' version is secure under the WNM and UNPRE properties of KEM.

\paragraph{The ``hash $\pw$, $(s,T)$ and $c$'' version.}
Note that the change in Game 2 (which requires WNM) and the change in Game 11 (which require UNPRE) are unrelated to each other. Therefore, if we define $\tau$ as $\Htag(\pw,s,T,c,K)$, then only Game 11 needs to change, and the protocol is secure under the UNPRE property of KEM.

\subsection{The ``Hash $pk$ and $(s,T)$'' Version}
\label{sec:hash-pk-st}
We now consider the variant where $\pw$, but not $pk$, is further removed from $\Htag$ inputs, i.e., $\tau$ is defined as $\Htag(pk,s,T,K)$. \Cref{tab:game-summary} suggests that Games 2 and 13 in \Cref{sec:games} need to (essentially) change. In fact the simulator also needs to change a bit, which we start from.

\subsubsection{Changes to the Simulator}
The only changes to the simulator $\sim$ appear when $\adv$ sends $(c^*,\tau^*)$ to $P$ and $\sim$ needs to extract a password guess $\pw^*$ from it. In the ``hash everything'' version (\Cref{sec:simulator}), $\tau^*$ is computed as $\Htag(\pw^*,pk^*,s,T,c^*,K^*)$, so $\sim$ can extract $\pw^*$ directly from $\adv$'s $\Htag$ queries and \emph{check its consistency with $pk^*,c^*,K^*$} (see the last paragraph of \Cref{sec:sim-intuition}). Here, $\tau^* = \Htag(pk^*,s,T,K^*)$, which prevents $\sim$ from reading off $\pw^*$ from $\adv$'s $\Htag$ queries. However, the inclusion of $pk^*$ still allows $\sim$ to find the corresponding $\langle\pw^*,pk^*,sk^*\rangle$ record, from which $\sim$ can recover $\pw^*$ and check for consistency. Still, there is a negligible chance that there are multiple $\langle\pw^*,pk^*,sk^*\rangle$ records for the same $pk^*$, which we must rule out.

We show the new simulator in \Cref{fig:oeke-sim-hash-pk-st}. (Only the steps after receiving $(c^*,\tau^*)$ are included.)

\subsubsection{Changes in the Games}

\paragraph{Change in Game 2.}
\begin{claim}
\label{clm:htag3}
Suppose the WNM property holds for the KEM scheme. In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, the probability that $P'$ queries $\Htag(pk,s,T,K)$ is negligible.
\end{claim}
$P'$ makes a single $\Htag$ query, on $(pk',s^*,T^*,K')$. If $(s^*,T^*) \neq (s,T)$, then of course $P'$'s query is not $\Htag(pk,s,T,K)$. Now assume $(s^*,T^*) = (s,T)$, which implies $\pw \neq \pw' \lor c^* \neq c$ by the condition of the game.
\begin{itemize}
    \item If $\pw \neq \pw'$, we have
      \begin{gather*}
      pk = \frac{T}{R} = \frac{T}{H_1(\pw,r)} = \frac{T}{H_1(\pw,s \oplus t)}, \\
      pk' = \frac{T}{R'} = \frac{T}{H_1(\pw',r')} = \frac{T}{H_1(\pw',s \oplus t')}.
      \end{gather*}
      Since $\pw \neq \pw'$, $H_1(\pw,s \oplus t)$ and $H_1(\pw',s \oplus t')$ are independent of each other, so $\Pr[pk = pk'] = \Pr[H_1(\pw',s \oplus t') = pk] \leq (q_1+1)/|\group|$.
  \item If $\pw = \pw' \land c^* \neq c$, then $\pw = \pw' \land (s^*,T^*) = (s,T)$, which implies that $pk = pk'$ by the definition of $pk$ and $pk'$. Then we can show that $\Pr[K = K']$ is negligible, via a reduction $\Adv_{\red_\text{WNM}}^\text{WNM}$ similar to that in the proof of \Cref{clm:htag2}. We have $\Pr[K = K'] \leq \Adv_{\red_\text{WNM}}^\text{WNM}$.
\end{itemize}

This yields the claim. The remaining argument is essentially identical to that in the old Game 2.

\paragraph{Change in Game 13.}
We first state the new Game 13, and then make the argument.

\medskip\noindent\emph{Game 13:} In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land (c^*,\tau^*) = (c,\tau'))$, find $\adv$'s $\Htag(pk^*,s,T,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. Otherwise there is a unique such $\Htag(pk^*,s,T,K^*)$ query, and
\begin{itemize}
  \item If there is more than one $\langle\pw^*,pk^*,sk^*\rangle$ record, abort.
  \item If there is no $\langle\pw^*,pk^*,sk^*\rangle$ record, set $SK := \bot$.
  \item If there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, but $\pw^* \neq \pw$, also set $SK := \bot$.
  \item If there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, and $\pw^* = \pw$, compute $K := \Decap(sk^*,c^*)$. If $K^* = K$ then set $SK := \Hkey(K)$, otherwise set $SK := \bot$.
\end{itemize}

If $\adv$ makes no $\Htag(\star,s,T,\star)$ query resulting in $\tau^*$, then in particular $\adv$ makes no $\Htag(pk,s,T,\star)$ query resulting in $\tau^*$, so both Game 12 and Game 13 set $SK := \bot$. The probability that $\adv$ makes two different $\Htag(\star,s,T,\star)$ queries both resulting in $\tau^*$ is upper-bounded by $\qtag(\qtag-1)/2^{2n+1}$. (This part does not change from the argument under the old Game 13.) Below we assume that $\adv$ makes a unique $\Htag(pk^*,s,T,K^*)$ query resulting in $\tau^*$:
\begin{itemize}
  \item We first upper-bound the probability that there is more than one $\langle\pw^*,pk^*,\allowbreak sk^*\rangle$ record for the same $pk^*$. Note that in each such record, $(pk^*,\star) \gets \Gen$ is freshly generated, so the probability that there is a collision in $pk^*$ is upper-bounded by ${q_2 \choose 2} \cdot H_\infty(pk : (pk,\star) \gets \Gen)$.
  \item If there is no $\langle\pw^*,pk^*,sk^*\rangle$ record, then both Game 12 and Game 13 set $SK := \bot$.
  \item Next, suppose there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, but $\pw^* \neq \pw$. If there is no $\langle\pw,pk,sk\rangle$ record, then both Game 12 and Game 13 set $SK := \bot$. If there is a $\langle\pw,pk,sk\rangle$ record, then $(pk,\star) \gets \Gen$ is freshly generated, so $\Pr[pk = pk^*] \leq q_2 \cdot H_\infty(pk : (pk,\star) \gets \Gen)$; and if $pk \neq pk^*$, then $\adv$ does not make an $\Htag(\pw,s,T,pk)$ query resulting in $\tau^*$ (because there is only one $\Htag$ query resulting in $\tau^*$, and it is on $pk^*$), so again both Game 12 and Game 13 set $SK := \bot$.
  \item Finally, if there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, and $\pw^* = \pw$, then Game 13 just runs $P$'s algorithm in Game 12, so these two games are identical.
\end{itemize}

In sum, we have
\[
\Dist_\env^{12,13} \leq \frac{q_2(q_2+1)}{2}\cdot H_\infty(pk : (pk,\star) \gets \Gen) + \frac{\qtag(\qtag-1)}{2^{2n+1}}.
\]

Overall, we conclude that the ``hash $pk$ and $(s,T)$'' version is secure under the WNM property of KEM.

\paragraph{The ``hash $pk$, $(s,T)$ and $c$'' version.}
If we define $\tau$ as $\Htag(pk,s,T,c,K)$, then the second bullet of \Cref{clm:htag3} is not needed, and the protocol can be proven secure without the WNM property of KEM (i.e., under the same assumptions as in the ``hash everything'' version).

\subsection{The ``Only Hash $(s,T)$'' Version}
\label{sec:hash-st}

Finally, we consider the variant where both $\pw$ and $pk$ (as well as $c$) are removed from $\Htag$ inputs, i.e., $\tau$ is defined as $\Htag(s,T,K)$. This is \emph{not} merely a combination of what we did while removing $\pw$ and while removing $pk$, and requires a new analysis. Again we will start from the changes to the simulator.

\subsubsection{Changes to the Simulator}
When $\pw$ is removed from $\Htag$ in \Cref{sec:hash-pk-st}, the simulator needs to change how it extracts $\adv$'s password guess $\pw^*$: it cannot read off $\pw^*$ from $\adv$'s $\Htag$ queries, and instead needs to read off $pk^*$ and then fetch the $\langle\pw^*,pk^*,sk^*\rangle$ record. Here, neither $\pw^*$ nor $pk^*$ can be read off from $\adv$'s $\Htag$ queries, and a new extraction strategy is needed. $\sim$ can only read off $K^*$, so it has to go over \emph{all} $\langle\pw^*,pk^*,sk^*\rangle$ records and check which one is consistent with $K^*$ and $c^*$. The new simulator is shown in \Cref{fig:oeke-sim-hash-st}.

\subsubsection{Changes in the Games}

\paragraph{Change in Game 2.}
\begin{claim}
\label{clm:htag4}
Suppose the KBIND and WNM properties hold for the KEM scheme. In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$, the probability that $P'$ queries $\Htag(s,T,K)$ is negligible.
\end{claim}
$P'$ makes a single $\Htag$ query, on $(s^*,T^*,K')$. If $(s^*,T^*) \neq (s,T)$, then of course $P'$'s query is not $\Htag(s,T,K)$. Now assume $(s^*,T^*) = (s,T)$, which implies $\pw \neq \pw' \lor c^* \neq c$ by the condition of the game.
\begin{itemize}
  \item If $\pw \neq \pw'$, by the proof of \Cref{clm:htag2}, $pk$ and $pk'$ are independent of each other (where $pk$ is honestly generated and $pk'$ is uniformly random). This suggests that we should upper-bound $\Pr[K = K'] = \Pr[\Decap(sk,c^*) = K']$ via a reduction $\red_\text{KBIND}$ to the KBIND property of KEM:
      \vspace{6pt} \\ % force \medskip in itemize environment
      \textbf{\underline{Reduction $\red_\mathrm{KBIND}$}:}
      \vspace{6pt} \\
      Sample $j \gets [q_2+1]$ as a guess that the $j$-th $H_2$ query (by either $\adv$ or $P'$) is $H_2(\pw',T)$. \\
      On input $(pk,pk',c,K')$,
      \begin{enumerate}
        \item Invoke $\env$. On $(\NewSession, sid, P, P', \pw, \text{``initiator''})$ from $\env$, run $P$'s algorithm to compute $(s,T)$, except that $\red_\mathrm{KBIND}$ uses its input $pk$ instead of $(pk,\star) \gets \Gen$. Send $(s,T)$ to $\adv$.
        \item On the $j$-th $H_2(\pw^*,T) =: t'$ query (by either $\adv$ or $\red_\mathrm{KBIND}$ itself on behalf of $P'$ --- see step 3 below), compute
            \[
            r' := s \oplus t', \quad R' := T/pk'.
            \]
            If $H_1(\pw^*,r')$ has already been defined and it is not $R'$, abort. Otherwise program $H_1(\pw^*,r') := R'$. \quad \textcolor[gray]{0.5}{// similar to how $H_1$ is programmed in the OW reduction in \Cref{sec:games}, Game 6}
        \item On $(\NewSession, sid, P', P, \pw', \text{``respondent''})$ from $\env$ and $(s^*,T^*)$ from $\adv$, if $\neg(\pw \neq \pw' \land (s^*,T^*) = (s,T))$, abort. Otherwise
            \begin{enumerate}
              \item If $\adv$ has not queried $H_2(\pw',T)$, make this query on behalf of $P'$.
              \item Compute $SK := \Hkey(K')$ and $\tau' := \Htag(s,T,K')$. Output $SK'$ to $\env$, and send $(c,\tau')$ to $\adv$.
            \end{enumerate}
        \item On $(c^*,\tau^*)$ from $\adv$, output $c^*$.
      \end{enumerate}
      Assuming $\Decap(sk,c^*) = K'$ and $\red_\text{KBIND}$'s guess of index $j$ is correct, the only difference between Game 1/2 and $\red_\text{KBIND}$'s simulation (until $\adv$ sends $c^*$) is that $\red_\text{KBIND}$ uses its input $pk'$ to program $H_1(\pw',r')$, and then uses its inputs $(c,K') \gets \Encap(pk')$ as the $P'$-to-$P$ message and compute $SK$ and $\tau'$. Since $(c,K')$ is also computed as $\Encap(pk')$ in Game 1/2, $\red_\text{KBIND}$ simulates Game 1/2 perfectly. Furthermore, $\red_\text{KBIND}$ wins the KBIND game. We have
  \item If $\pw = \pw' \land c^* \neq c$, then the argument in the proof of \Cref{clm:htag3} still holds, and $\Pr[K = K'] \leq \Adv_{\red_\text{WNM}}^\text{WNM}$.
\end{itemize}

Overall, we have
\[
\Pr[P'\text{ queries }\Htag(s,T,K)] \leq (q_2+1)\Adv_{\red_\text{KBIND}}^\text{KBIND} + \Adv_{\red_\text{WNM}}^\text{WNM},
\]
which yields the claim. The remaining argument is essentially identical to that in the old Game 2.

\paragraph{Change in Game 11.}
Game 10 and Game 11 differ only if $\adv$ does not make an $H_2(\pw,T)$ query, but makes an $\Htag(s,T,K^*)$ query resulting in $\tau^*$, where $K^* = \Decap(sk,c^*)$ for a freshly generated $sk$. By a reduction $\red_\text{UNPRE}$ that is essentially identical to the one in \Cref{sec:hash-pw-st}, we have
\[
\Dist_\env^{10,11} \leq \Adv_{\red_\text{UNPRE}}^\text{UNPRE}.
\]

\paragraph{Change in Game 13.}
We first state the new Game 13, and then make the argument.

\medskip\noindent\emph{Game 13:} In the case where $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land (c^*,\tau^*) = (c,\tau'))$, find $\adv$'s $\Htag(s,T,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. Otherwise there is a unique such $\Htag(s,T,K^*)$ query, and
\begin{itemize}
  \item For each $\langle\pw^*,pk^*,sk^*\rangle$ record, compute $K := \Decap(sk^*,c^*)$ and check if $K^* = K$. If there is more than one such record, abort.
  \item If there is no such $\langle\pw^*,pk^*,sk^*\rangle$ record, set $SK := \bot$.
  \item If there is a unique such $\langle\pw^*,pk^*,sk^*\rangle$ record, but $\pw^* \neq \pw$, also set $SK := \bot$.
  \item If there is a unique such $\langle\pw^*,pk^*,sk^*\rangle$ record, and $\pw^* = \pw$, set $SK := \Hkey(K)$.
\end{itemize}

If $\adv$ makes no $\Htag(s,T,\star)$ query resulting in $\tau^*$, then both Game 12 and Game 13 set $SK := \bot$. The probability that $\adv$ makes two different $\Htag(s,T,\star)$ queries both resulting in $\tau^*$ is upper-bounded by $\qtag(\qtag-1)/2^{2n+1}$. (This part does not change from the argument under the old Game 13.) Below we assume that $\adv$ makes a unique $\Htag(s,T,K^*)$ query resulting in $\tau^*$:
\begin{itemize}
  \item We first upper-bound the probability that there is more than one $\langle\pw^*,pk^*,\allowbreak sk^*\rangle$ record with $\Decap(sk^*,c^*) = K^*$. Note that in each such record, $(\star,sk^*) \gets \Gen$ is freshly generated, so a straightforward reduction to the ROB property of KEM, $\red_\text{ROB1}$, shows that the probability that the same $c^*$ decapsulates to the same $K^*$ under two freshly generated $sk^*$ is upper-bounded by ${q_2 \choose 2} \cdot \Adv_{\red_\text{ROB1}}^\text{ROB}$.
  \item If there is no $\langle\pw^*,pk^*,sk^*\rangle$ record with $\Decap(sk^*,c^*) = K^*$, then both Game 12 and Game 13 set $SK := \bot$.
  \item Next, suppose there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record with $\Decap(sk^*,c^*) = K^*$, but $\pw^* \neq \pw$. If there is no $\langle\pw,pk,sk\rangle$ record, then both Game 12 and Game 13 set $SK := \bot$. If there is a $\langle\pw,pk,sk\rangle$ record, then $(\star,sk) \gets \Gen$ is freshly generated, so a straightforward reduction to the ROB property of KEM, $\red_\text{ROB2}$, shows that the probability that the same $c^*$ decapsulates to the same $K^*$ under both $sk$ and $sk^*$ is upper-bounded by $q_2 \cdot \Adv_{\red_\text{ROB2}}^\text{ROB2}$. If this does not happen, then again both Game 12 and Game 13 set $SK := \bot$.
  \item Finally, if there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, and $\pw^* = \pw$, then both Game 12 and Game 13 set $SK := \Hkey(K)$.
\end{itemize}

In sum, we have
\[
\Dist_\env^{12,13} \leq \frac{q_2(q_2-1)}{2} \cdot \Adv_{\red_\text{ROB1}}^\text{ROB1} + q_2 \cdot \Adv_{\red_\text{ROB2}}^\text{ROB2} + \frac{\qtag(\qtag-1)}{2^{2n+1}}.
\]

Overall, we conclude that the ``only hash $(s,T)$'' version is secure under the KBIND, WNM, UNPRE and ROB properties of KEM.

\paragraph{The ``hash $(s,T)$ and $c$'' version.}
If we define $\tau$ as $\Htag(s,T,c,K)$, then the second bullet of \Cref{clm:htag4} is not needed, and the protocol can be proven secure without the WNM property of KEM (but we still need KBIND, UNPRE and ROB).

\section{Security Analyses of Other Versions (II): Removing $(s,T)$ from the Hash}
\label{app:no-st}
\subsection{The ``Hash $\pw$'' Version}
\label{sec:hash-pw}
We now turn to the variant where $pk$ is further removed from $\Htag$ inputs, i.e., $\tau$ is defined as $\Htag(\pw,K)$. \Cref{tab:game-summary} suggests that Games 2 and 11 in \Cref{sec:games} need to (essentially) change. In fact the simulator also needs to change quite a bit, which we will start from.

\subsubsection{Changes to the Simulator}
The only changes to the simulator $\sim$ appear when $\adv$ sends $(c^*,\tau^*)$ to $P$ and $\sim$ needs to extract a password guess $\pw^*$ from it. In the ``hash everything'' version (\Cref{sec:simulator}), $\tau^*$ is computed as $\Htag(\pw^*,pk^*,c^*,K^*)$, so $\sim$ can extract $\pw^*$ from $\adv$'s $\Htag$ queries and \emph{check its consistency with $pk^*,c^*,K^*$} (see the last paragraph of \Cref{sec:sim-intuition}). Here, $\tau^* = \Htag(\pw^*,K^*)$, but the inclusion of $\pw^*$ still allows $\sim$ to find the corresponding $\langle\pw^*,pk^*,sk^*\rangle$ record and check for consistency.

However, a more sinister issue emerges when we remove $pk^*$ from $\Htag$: In the case where the two parties' passwords match, $\adv$ can modify $c$ and force the two parties' session keys to be the same --- without making \emph{any} $\Htag$ query! The easiest way to see this is to work out a concrete version for OEKE based on the Diffie-Hellman KEM, which we show in \Cref{fig:attack-no-pk}.

\begin{figure}[H]
\pseudocodeblock[codesize=\scriptsize]{
P(\pw) \< \< \adv \< \< P'(\pw) \\ 
\< \sendmessageright*[1.5cm]{s,T} \< \< \< \\
\< \< X := \mathsf{2F}^{\text{-}1}(\pw,(s,T)) \< \< \\
\< \< (s^*,T^*) := \mathsf{2F}(\pw,X^2) \< \< \\
\< \< \<\sendmessageright*[1.5cm]{s^*,T^*} \< \\
\< \sendmessageleft*[1.5cm]{Y^2,\tau'} \< \< \sendmessageleft*[1.5cm]{Y,\tau'} \< \< \\
\text{output} \< \< \< \< \text{output} \\
\Hkey((Y^2)^x) = \Hkey(g^{2xy}) \< \< \< \< \Hkey((X^2)^y) = \Hkey(g^{2xy})
}
\caption{New attack on the ``hash $\pw$'' version}
\label{fig:attack-no-pk}
\end{figure}

In this attack, the two honest parties' passwords $\pw$ are the same. Assuming $\adv$ knows it, it can decrypt $P$'s message to recover $X = g^x$, and then re-encrypt $X^2$ under the same $\pw$; this causes $P'$ to output $SK' = \Hkey(g^{2xy})$ and send $(Y = g^y, \tau' = \Htag(\pw,g^{2xy}))$. Then $\adv$ sends $(Y^2,\tau')$ to $P$, causing $P$'s check to pass and $P$ will also output $\Hkey(g^{2xy})$. (This is not an issue if $pk$ is included in $\Htag$, since $P$ would compute $\tau = \Htag(\dots,Y^2,\dots) \neq \Htag(\dots,Y,\dots) = \tau'$ and its check would fail.)

In fact, it is a stretch to call this an attack, since the only thing $\adv$ does is to modify some protocol messages and cause $P$ and $P'$ to output the same session key --- which $\adv$ \emph{cannot} compute. Still, this scenario is not covered by the previous simulator. It should be simulated as follows: When $\adv$ decrypts $(s,T)$ under $\pw$, $\sim$ records $\langle\pw,pk,sk\rangle$ (although there might be a large number of such records, and $\sim$ does not know which is the ``right'' one at this point). Then when $\adv$ sends $(s^*,T^*)$ to $P'$, $\sim$ extracts $\pw$, which allows $\sim$ to retrieve the ``right'' record $\langle\pw,pk,sk\rangle$; $\sim$ also computes $K'$ while simulating the $P'$ side. Later, when $\adv$ sends $(c^*,\tau')$, $\sim$ can tell if $\adv$ is performing the aforementioned attack by checking if $\Decap(sk,c^*) = K'$.

We show the new simulator in \Cref{fig:oeke-sim-hash-pw}. (Only the steps after receiving $(c^*,\tau^*)$ are included.)

\subsubsection{Changes in Game Hops}
\label{sec:new-games}
Note that \Cref{clm:htag} in Game 2 does not hold anymore, exactly due to what the adversary can do in \Cref{fig:attack-no-pk} ($P$ does not reject even though $\adv$ makes no $\Htag$ query whatsoever). As such, Game 2 needs to change essentially. Since Game 2 is almost at the beginning of the sequence of games, this affects some subsequent games and we need to proceed carefully. (Most of it is clerical work verifying that the previous argument still goes through; the parts that need to change essentially are shown in {\color{blue}blue}.) We start from describing the new Game 2.

\medskip\noindent\emph{Game 2:} In the case where {\color{blue}$\pw \neq \pw' \lor ((s^*,T^*) = (s,T) \land c^* \neq c)$}, find $\adv$'s $\Htag(\pw,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. If there is a unique such query, do the following:
\begin{itemize}
  \item If $K^* = K$, set $SK := \Hkey(K)$ (i.e., there is no change from Game 1).
  \item If $K^* \neq K$, set $SK := \bot$.
\end{itemize}

Furthermore, in the case where $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, do the same as above, except that we now find $\adv$ {\color{blue}\emph{or $P'$}}'s $\Htag(\pw,K^*)$ query resulting in $\tau^*$.

The difference between the new Game 2 and the old one is that we now have two separate cases (which, combined, are identical to the case in the old Game 2): in the first case we proceed as in the old Game 2, whereas in the second case we cannot due to the ``attack'' in \Cref{fig:attack-no-pk}; instead, in the second case we need to include $P'$'s $\Htag$ query as if it were $\adv$'s.

\begin{claim}
\label{clm:htag3}
Suppose the WNM property holds for the KEM scheme. In the first case above, the probability that $P'$ queries $\Htag(\pw,K)$ is negligible.
\end{claim}

The proof of this claim is essentially part of the proof of \Cref{clm:htag2}. $P'$ makes a single $\Htag$ query, on $(\pw',K')$. There are two cases:
\begin{itemize}
  \item If $\pw \neq \pw'$, then of course $P'$'s query is not $\Htag(\pw,K)$.
  \item If $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* \neq c$, then the argument in the proof of \Cref{clm:htag2} still goes through, so $\Pr[K = K'] \leq \Adv_{\red_\text{WNM}}^\text{WNM}$.
\end{itemize}

This yields the claim. The remaining argument is essentially identical to that in the old Game 2.

\medskip\noindent\emph{Games 3--4:} These two games deal with the case where $\pw = \pw' \land (s^*,T^*) = (s,T)$. This falls into the first case of Game 2, and in this case there is no difference between the old and new versions of Game 2. As such, the reductions still go through: they abort if $\pw = \pw' \land (s^*,T^*) = (s,T)$ does not hold, and if $\pw = \pw' \land (s^*,T^*) = (s,T)$, they can still simulate the games by checking $\adv$'s $\Htag$ queries. Crucially, the reductions do not need to check $P'$'s $\Htag$ query (which they cannot do), since in this case we have defined Game 2 so that it only checks $\adv$'s $\Htag$ queries.

\medskip\noindent\emph{Game 5:} This game involves a reduction to the UNI-PK property of KEM. The reduction works essentially because $sk'$ is not used anywhere in the game (so the reduction can embed its challenge as $pk'$ without knowing $sk'$), and this fact still holds even though we need to additionally check $P'$'s $\Htag$ queries in some cases now. So, this reduction still goes through.

\medskip\noindent\emph{Games 6--8:} These three games deal with the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T)$. This again falls into the first case of Game 2, so the reductions in Games 6 and 7 still go through; Game 8 involves a statistical argument which does not need to change either.

\medskip\noindent\emph{Games 9--10:} These two games involve statistical arguments only, and they have nothing to do with whether the games need to check $P'$'s $\Htag$ queries.

\medskip\noindent\emph{Game 11:} The argument for this game needs to change essentially (see \Cref{tab:game-summary}). {\color{blue}Game 10 and Game 11 differ if $\adv$ does not make an $H_2(\pw,T)$ query, but $\adv$ \emph{or $P'$} makes an $\Htag(\pw,K^*)$ query resulting in $\tau^*$, where $K^* = \Decap(sk,c^*)$ for a freshly generated $sk$. A straightforward reduction to the UNPRE property of KEM, $\red_\text{UNPRE1}$, shows that
\[
\Dist_\env^{10,11} \leq \Adv_{\red_\text{UNPRE1}}^{\text{UNPRE}}.
\]}

\noindent\emph{Game 12:} This game should read as follows now: In the case where $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c \land \tau^* \neq \tau'$, do what's in the $\pw \neq \pw' \lor ((s^*,T^*) = (s,T) \land c^* \neq c)$ case. That is, first find the $\langle\pw,\star,\star\rangle$ record (as per Game 11), then find $\adv$'s $\Htag(\pw,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. If there is a unique such query, do the following:
\begin{itemize}
  \item If $K^* = K$, set $SK := \Hkey(K)$.
  \item If $K^* \neq K$, set $SK := \bot$.
\end{itemize}

In Game 11 $SK$ is outright set to $\bot$, whereas here it checks $\adv$'s $\Htag$ queries \emph{but does not check $P'$'s $\Htag$ query}. Therefore, this game hop has nothing to do with what's different in the new Game 2, and the previous argument still goes through.

\medskip\noindent\emph{Game 13:} This game should now extract $\pw^*$ from $\adv$ \emph{or $P'$}'s $\Htag$ query, based on the cases in Game 2 and Game 12. For completeness, we include a full description of the game.

In the case where $\pw \neq \pw' \lor ((s^*,T^*) = (s,T) \land (c^* \neq c \lor (c^* = c \land \tau^* \neq \tau')))$, find $\adv$'s $\Htag(\pw^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. Otherwise there is a unique such $\Htag(\pw^*,K^*)$ query, and
\begin{itemize}
  \item If $\pw^* \neq \pw$, set $SK := \bot$.
  \item If $\pw^* = \pw$, run the remaining part of $P'$'s algorithm except that we now use $\pw^*$ instead of $\pw$ --- that is, if there is no $\langle\pw^*,pk^*,sk^*\rangle$ record then set $SK := \bot$; otherwise compute $K := \Decap(sk^*,c^*)$, and if $K^* = K$ then set $SK := \Hkey(K)$, otherwise set $SK := \bot$.
\end{itemize}

Furthermore, in the case where $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, do the same as above, except that we now find $\adv$ {\color{blue}\emph{or $P'$}}'s $\Htag(\pw^*,K^*)$ query resulting in $\tau^*$.

The previous statistical argument does not really change.

\medskip\noindent\emph{Games 14--16:} The statistical arguments for these three games do not need to change.

{\color{blue}
\medskip\noindent\emph{New game 1:} Now that $(\pw')^*$ --- the password guess extracted from $(s^*,T^*)$ --- is defined, we can finally add (part of) what's in step 2 of \Cref{fig:oeke-sim-hash-pw}. Concretely, we change the game as follows:

In the case where $(s^*,T^*) \neq (s,T) \land (\pw')^* = \pw = \pw' \land \tau^* = \tau'$, there is a record $\langle\pw,\star,sk\rangle$, and $\Decap(sk,c^*) = K'$, set $SK := \Hkey(K')$. (In some sense this is the most crucial new game, as it covers the scenario in \Cref{fig:attack-no-pk} --- which is why we need to make all these changes to begin with. It also corresponds to step 2(c)(ii) in \Cref{fig:oeke-sim-hash-pw}. We will deal with step 2(c)(i) later.)

Since $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, Game 16 finds $\adv$ or $P'$'s $\Htag$ query resulting in $\tau^*$ (see the description of Game 13). $P'$ makes a singe query to $\Htag$, on input $(\pw',K')$; and the query result is $\tau' = \tau^*$. Therefore, Game 16 finds this query, and does the following (again see the description of Game 13):
\begin{enumerate}
  \item Check if $\pw' = \pw$, which is indeed the case.
  \item Find the $\langle\pw,pk,sk\rangle$ record.
  \item Compute $K := \Decap(sk,c^*)$ and check if $K = K'$, which is again the case by the condition of this game.
  \item Set $SK := \Hkey(K) (= \Hkey(K'))$.
\end{enumerate}
This entire process in Game 16 eventually results in setting $SK := \Hkey(K')$, which exactly matches the new game. Thus, there is no difference. We have
\[
\Dist_\env^{16,\text{new}1} = 0.
\]

\noindent\emph{New game 2:} In the case where $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, we find $\adv$ \emph{or $P'$}'s $\Htag(\pw^*,K^*)$ query resulting in $\tau^*$ (except that in new game 1's condition, new game 1 already shortcuts it to $SK := \Hkey(K')$ without trying to find any $\Htag$ query). In this game, we do not check $P'$'s $\Htag$ query anymore, and only check $\adv$'s $\Htag$ query.

There are the following cases:
\begin{itemize}
  \item If $\tau^* \neq \tau'$, then of course $P'$'s $\Htag$ query does not result in $\tau^*$ (it results in $\tau'$), so we can safely drop $P'$'s $\Htag$ query.
  \item If $(\pw')^* \neq \pw' \land \tau^* = \tau'$, then $P'$'s $\Htag(\pw',K')$ query results in $\tau^*$, and we need to argue that this query can be dropped from the check.
      \begin{itemize}
        \item If $\adv$ queries $\Htag(\pw',K')$, then new game 2 still finds this query, so new game 1 and new game 2 are identical.
        \item If $\adv$ does not query $\Htag(\pw',K')$, then new game 2 sets $SK := \bot$. We argue that new game 1 also sets $SK := \bot$ except with negligible probability. New game 1 finds $P'$'s $\Htag(\pw',K')$ query, and sets $SK := \bot$ unless $\Decap(sk,c^*) = K'$. But note that $sk$ is freshly generated, so a straightforward reduction to the UNPRE property of KEM, $\red_\text{UNPRE2}$, shows that $\Decap(sk,c^*) = K'$ happens with negligible probability (cf. the reduction in Game 11).
      \end{itemize}
  \item If $(\pw')^* = \pw' \land \tau^* = \tau'$, but there is no $\langle\pw,\star,sk\rangle$ record, then both new game 1 and new game 2 outright sets $SK := \bot$, regardless of $\Htag$ queries by $\adv$ or $P'$. Thus, the two games are identical.
  \item If $(\pw')^* = \pw' \land \tau^* = \tau'$, but $\Decap(sk,c^*) \neq K'$, then new game 1 finds $P'$'s $\Htag(\pw',K')$ query and then sets $SK := \bot$ because the check $\Decap(sk,c^*) \stackrel{?}{=} K'$ does not pass, and new game 2 also sets $SK := \bot$ because it either fails to find $\adv$'s $\Htag(\pw',K')$ query, or it finds such a query but then sets $SK := \bot$ because the check $\Decap(sk,c^*) \stackrel{?}{=} K'$ does not pass. Thus, the two games are identical.
\end{itemize}

The remaining case is that $(\pw')^* = \pw' \land \tau^* = \tau'$, there is a $\langle\pw,\star,sk\rangle$ record, and $\Decap(sk,c^*) = K'$. This is exactly the condition in new game 1, and new game 1 outright sets $SK := \Hkey(K')$ without trying to find any $\Htag$ query (by $\adv$ or $P'$), so the game change does not affect this case. Overall, we conclude that
\[
\Dist_\env^{\text{new}1,\text{new}2} \leq \Adv_{\red_\text{UNPRE2}}^{\text{UNPRE}}.
\]}

\noindent\emph{Game 17:} Note that after new game 2, we never try to find $P'$'s $\Htag$ query anymore: in some cases we try to find $\adv$'s $\Htag$ query, and in other cases we outright set $SK$ to be a certain value or $\bot$. Therefore, the reduction still goes through (it can simulate the $P$ side honestly).\footnote{New game 1 and new game 2 --- which drop $P'$'s $\Htag$ query from the check --- have to be inserted before Game 17, since the reduction in Game 17 cannot check $P'$'s $\Htag$ query (the query is $\Htag(\pw',K')$, and the reduction does not know $K'$).}

{\color{blue}
\medskip\noindent\emph{New game 3:}
In the case where $(s^*,T^*) \neq (s,T) \land (\pw')^* = \pw' \neq \pw \land \tau^* = \tau'$, there is a record $\langle\pw',\star,sk\rangle$, and $\Decap(sk,c^*) = K'$, set $SK := \bot$. (This corresponds to step 2(c)(i) in \Cref{fig:oeke-sim-hash-pw}.)

In this case Game 17 tries to find $\adv$'s (unique) $\Htag(\pw^*,K^*)$ query resulting in $\tau^*$, and checks if $\pw^* = \pw$ (see Game 13). But $\tau^* = \tau' = \Htag(\pw',K')$, so $\pw^* = \pw' \neq \pw$ and the check never passes --- that is, both Game 17 and new game 3 set $SK := \bot$. We have
\[
\Dist_\env^{17,\text{new}3} = 0.
\]

\paragraph{Summary.}
We have made the following essential changes to the games:
\begin{itemize}
  \item In the case where $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, Game 2 tries to find $P'$'s $\Htag$ query resulting in $\tau^*$ (in addition to $\adv$'s query).
  \item In the case where $(s^*,T^*) \neq (s,T) \land (\pw')^* = \pw = \pw' \land \tau^* = \tau'$, there is a record $\langle\pw,\star,sk\rangle$, and $\Decap(sk,c^*) = K'$, new game 1 outright sets $SK := \Hkey(K')$ (without trying to find any $\Htag$ query by $\adv$ or $P'$).
  \item In the case where $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, new game 2 drops the check on $P'$'s $\Htag$ query introduced in Game 2 (except in new game 1's condition, where all checks on $\Htag$ queries have already been dropped).
  \item In the case where $(s^*,T^*) \neq (s,T) \land (\pw')^* = \pw' \neq \pw \land \tau^* = \tau'$, there is a record $\langle\pw',\star,sk\rangle$, and $\Decap(sk,c^*) = K'$, new game 3 outright sets $SK := \bot$.
\end{itemize}

Overall, the net changes are that in the case where $(s^*,T^*) \neq (s,T) \land (\pw')^* = \pw' \land \tau^* = \tau'$, there is a record $\langle\pw',\star,sk\rangle$, and $\Decap(sk,c^*) = K'$,
\begin{itemize}
  \item If $(\pw')^* = \pw$, then we set $SK := \Hkey(K')$.
  \item If $(\pw')^* \neq \pw$, then we set $SK := \bot$.
\end{itemize}
This exactly matches step 2 of \Cref{fig:oeke-sim-hash-pw}, so new game 3 is identical to the ideal world.} This completes the proof.

\subsection{The ``Hash $pk$'' Version}
We now consider the variant where $\pw$, but not $pk$, is further removed from $\Htag$ inputs, i.e., $\tau$ is defined as $\Htag(pk,K)$. \Cref{tab:game-summary} suggests that Games 2 and 13 in \Cref{sec:games} need to (essentially) change; however, we will (again) start from the changes to the simulator.

\subsubsection{Changes to the Simulator}
Similar to \Cref{sec:hash-pw}, once we remove $pk$ from $\Htag$, $\adv$ can force the two parties' session keys to be the same without performing a ``real'' attack. However, this time the two parties' passwords are different (and $\adv$ needs to know both of them), and $\adv$ should forward $(c,\tau')$ from $P'$ without modifications. We show this attack in \Cref{fig:attack-no-pw}.

\begin{figure}[H]
\pseudocodeblock{
P(\pw) \< \< \adv \< \< P'(\pw') \\ 
\< \sendmessageright*[1.5cm]{s,T} \< \< \< \\
\< \< pk := \mathsf{2F}^{\text{-}1}(\pw,(s,T)) \< \< \\
\< \< (s^*,T^*) := \mathsf{2F}(\pw',pk) \< \< \\
\< \< \<\sendmessageright*[1.5cm]{s^*,T^*} \< \\
\< \sendmessageleft*[1.5cm]{c,\tau'} \< \< \sendmessageleft*[1.5cm]{c,\tau'} \< \< \\
\text{output }SK' \< \< \< \< \text{output }SK'
}
\caption{New attack on the ``hash $pk$'' version}
\label{fig:attack-no-pw}
\end{figure}

In this attack, the two honest parties' passwords $\pw$ and $\pw'$ may or may not be the same. Assuming $\adv$ knows both, it can decrypt $P$'s message to recover $pk$, and then re-encrypt $pk$ under $\pw'$; this ensures that $P'$ will recover the same $pk$ as on the $P$ side, and use it to compute $c,K',\tau'$ (and eventually $SK'$). But then if $\adv$ forwards $(c,\tau')$ without any modification, $P$ will decapsulate $c$ to the same $K'$ (because it uses the same $pk$), and its output key will be the same as $SK'$ on the $P'$ side. (This is not an issue if $\pw$ is included in $\Htag$, since forwarding $\tau' = \Htag(\pw',\dots) \neq \Htag(\pw,\dots)$ to $P$ would make $P$'s check fail.)

This is (again) a new situation not covered by the previous simulator. It should be simulated as follows: When $\adv$ decrypts $(s,T)$ under $\pw$, $\sim$ records $\langle\pw,pk,sk\rangle$ (although there might be a large number of such records, and $\sim$ does not know which is the ``right'' one at this point). Then when $\adv$ sends $(s^*,T^*)$ to $P'$, $\sim$ extracts $\pw'$, \emph{which also gives a $pk'$}. Later, when $\adv$ forwards $(c,\tau')$, $\sim$ knows that $\adv$ is performing the aforementioned attack, and checks which $\langle\pw,pk,sk\rangle$ record matches $pk'$. In this way $\sim$ can extract the password guess $\pw$ on the $P$ side.

We show the new simulator in \Cref{fig:oeke-sim-hash-pk}. (Only the steps after receiving $(c^*,\tau^*)$ are included.)

\subsubsection{Changes in Game Hops}
The changes in game hops again need to start from Game 2. The structure of the proof is very similar to \Cref{sec:new-games}; however, for the sake of completeness we still include a proof on the same level of details as in \Cref{sec:new-games}. (This involves copying a large chunk of arguments verbatim from \Cref{sec:new-games}.)

\medskip\noindent\emph{Game 2:} In the case where {\color{blue}$\pw = \pw' \lor (s^*,T^*) = (s,T) \lor c^* \neq c$} yet $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c)$ (i.e., excluding the condition of Game 1), find $\adv$'s $\Htag(pk,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. If there is a unique such query, do the following:
\begin{itemize}
  \item If $K^* = K$, set $SK := \Hkey(K)$ (i.e., there is no change from Game 1).
  \item If $K^* \neq K$, set $SK := \bot$.
\end{itemize}

Furthermore, in the case where $\pw \neq \pw' \land (s^*,T^*) \neq (s,T) \land c^* = c$, do the same as above, except that we now find $\adv$ {\color{blue}\emph{or $P'$}}'s $\Htag(pk,K^*)$ query resulting in $\tau^*$.
\begin{claim}
\label{clm:htag3}
Suppose the WNM property holds for the KEM scheme. In the first case above, the probability that $P'$ queries $\Htag(pk,K)$ is negligible.
\end{claim}
$P'$ makes a single $\Htag$ query, on $(pk',K')$. There are the following cases:
\begin{itemize}
  \item If $c^* \neq c$, we claim that $pk = pk' \land K = K'$ happens with negligible probability. Suppose $pk = pk'$. Then we can reduce the event $K = K'$ to the WNM property of KEM, via a reduction $\Adv_{\red_\text{WNM}}^\text{WNM}$ similar to that in the proof of \Cref{clm:htag2}. We have $\Pr[pk = pk' \land K = K'] \leq \Adv_{\red_\text{WNM}}^\text{WNM}$.
  \item If $c^* = c$, then $\pw = \pw' \lor (s^*,T^*) = (s,T)$ (by the condition of this game) yet $\neg(\pw = \pw' \land (s^*,T^*) = (s,T))$ (otherwise we fall into the condition of Game 1).
      \begin{itemize}
        \item If $\pw = \pw' \land (s^*,T^*) \neq (s,T)$, then the argument in the proof of \Cref{clm:htag} still goes through, so $\Pr[pk = pk'] \leq (q_1+1)/|\group|$.
        {\color{blue}\item If $\pw \neq \pw' \land (s^*,T^*) = (s,T)$, this case is not covered by previous proofs and needs a new argument. In this case we have
            \begin{gather*}
            pk = \frac{T}{R} = \frac{T}{H_1(\pw,r)} = \frac{T}{H_1(\pw,s \oplus t)}, \\
            pk' = \frac{T}{R'} = \frac{T}{H_1(\pw',r')} = \frac{T}{H_1(\pw',s \oplus t')}.
            \end{gather*}
            Since $\pw \neq \pw'$, $H_1(\pw,s \oplus t)$ and $H_1(\pw',s \oplus t')$ are independent of each other, so $\Pr[pk = pk'] = \Pr[H_1(\pw,s \oplus t) = H_1(\pw',s \oplus t')] \leq (q_1+1)/|\group|$.}
      \end{itemize}
\end{itemize}

This yields the claim. The remaining argument is essentially identical to that in the old Game 2.

\medskip\noindent\emph{Games 3--4:} These two games deal with the case where $\pw = \pw' \land (s^*,T^*) = (s,T)$. This falls into the first case of Game 2, and in this case there is no difference between the old and new versions of Game 2. As such, the reductions still go through: they abort if $\pw = \pw' \land (s^*,T^*) = (s,T)$ does not hold, and if $\pw = \pw' \land (s^*,T^*) = (s,T)$, they can still simulate the games by checking $\adv$'s $\Htag$ queries. Crucially, the reductions do not need to check $P'$'s $\Htag$ query (which they cannot do), since in this case we have defined Game 2 so that it only checks $\adv$'s $\Htag$ queries.

\medskip\noindent\emph{Game 5:} This game involves a reduction to the UNI-PK property of KEM. The reduction works essentially because $sk'$ is not used anywhere in the game (so the reduction can embed its challenge as $pk'$ without knowing $sk'$), and this fact still holds even though we need to additionally check $P'$'s $\Htag$ queries in some cases now. So, this reduction still goes through.

\medskip\noindent\emph{Games 6--8:} These three games deal with the case where $\pw \neq \pw' \land (s^*,T^*) = (s,T)$. This again falls into the first case of Game 2, so the reductions in Games 6 and 7 still go through; Game 8 involves a statistical argument which does not need to change either.

\medskip\noindent\emph{Games 9--10:} These two games involve statistical arguments only, and they have nothing to do with whether the games need to check $P'$'s $\Htag$ queries.

\medskip\noindent\emph{Game 11:} This game hop goes through for essentially the same reason as in \Cref{sec:games}. The difference is that now Game 10 and Game 11 differ if $\adv$ does not make an $H_2(\pw,T)$ query, but $\adv$ \emph{{\color{blue}or $P'$}} makes an $\Htag(pk,K^*)$ query resulting in $\tau^*$. However, $pk$ is freshly generated by $P$, independent of \emph{both} $\adv$ and $P'$'s views. Thus, the distinguishing advantage between Game 10 and Game 11 is still upper-bounded by $H_\infty(pk : (pk,\star) \gets \Gen)$.

\medskip\noindent\emph{Game 12:} This game should read as follows now: In the case where $\pw = \pw' \land (s^*,T^*) = (s,T) \land c^* = c \land \tau^* \neq \tau'$, do what's in the $\pw \neq \pw' \lor ((s^*,T^*) = (s,T) \land c^* \neq c)$ case. That is, first find the $\langle\pw,\star,\star\rangle$ record (as per Game 11), then find $\adv$'s $\Htag(pk,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. If there is a unique such query, do the following:
\begin{itemize}
  \item If $K^* = K$, set $SK := \Hkey(K)$.
  \item If $K^* \neq K$, set $SK := \bot$.
\end{itemize}

In Game 11 $SK$ is outright set to $\bot$, whereas here it checks $\adv$'s $\Htag$ queries \emph{but does not check $P'$'s $\Htag$ query}. Therefore, this game hop has nothing to do with what's different in the new Game 2, and the previous argument still goes through.

\medskip\noindent\emph{Game 13:} This game, as well as its argument, need to change quite a bit. We first include a full description of the game.

In the case where {\color{blue}$\pw = \pw' \lor (s^*,T^*) = (s,T) \lor (c^*,\tau^*) \neq (c,\tau')$ yet $\neg(\pw = \pw' \land (s^*,T^*) = (s,T) \land (c^*,\tau^*) = (c,\tau'))$}, find $\adv$'s $\Htag(pk^*,K^*)$ query resulting in $\tau^*$. If there is none or more than one, set $SK := \bot$. Otherwise there is a unique such $\Htag(pk^*,K^*)$ query, and
\begin{itemize}
  \item If there is no {\color{blue}or more than one} $\langle\pw^*,pk^*,sk^*\rangle$ record, set $SK := \bot$.
  \item If there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, but $\pw^* \neq \pw$, also set $SK := \bot$.
  \item If there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, and $\pw^* = \pw$, compute $K := \Decap(sk^*,c^*)$. If $K^* = K$ then set $SK := \Hkey(K)$, otherwise set $SK := \bot$.
\end{itemize}

Furthermore, in the case where {\color{blue}$\pw \neq \pw' \land (s^*,T^*) \neq (s,T) \land (c^*,\tau^*) = (c,\tau')$}, do the same as above, except that we now find $\adv$ {\color{blue}\emph{or $P'$}}'s $\Htag(pk^*,K^*)$ query resulting in $\tau^*$.

{\color{blue}We first upper-bound the probability that there is more than one $\langle\pw^*,pk^*,sk^*\rangle$ record for the same $pk^*$. Note that in each such record, $(pk^*,\star) \gets \Gen$ is freshly generated, so the probability that there is a collision in $pk^*$ is upper-bounded by ${q_2 \choose 2} \cdot H_\infty(pk : (pk,\star) \gets \Gen)$.

Next, suppose there is a unique $\langle\pw^*,pk^*,sk^*\rangle$ record, but $\pw^* \neq \pw$. If there is no $\langle\pw,pk,sk\rangle$ record, then both Game 12 and Game 13 set $SK := \bot$. If there is a $\langle\pw,pk,sk\rangle$ record, then $(pk,\star) \gets \Gen$ is freshly generated, so $\Pr[pk = pk^*] \leq q_2 \cdot H_\infty(pk : (pk,\star) \gets \Gen)$; and if $pk \neq pk^*$, then $\adv$ does not make an $\Htag(\pw,pk)$ query resulting in $\tau^*$ (because there is only one $\Htag$ query resulting in $\tau^*$, and it is on $pk^*$), so again both Game 12 and Game 13 set $SK := \bot$.}

Assuming neither of the above happens, the previous statistical argument does not really change.

\medskip\noindent\emph{Games 14--16:} The statistical arguments for these three games do not need to change.

{\color{blue}
\medskip\noindent\emph{New game 1:} Now that $(\pw')^*$ --- the password guess extracted from $(s^*,T^*)$ --- is defined, we can finally add (part of) what's in step 2 of \Cref{fig:oeke-sim-hash-pk}. Concretely, we change the game as follows:

In the case where $(\pw')^* = \pw' \land (s^*,T^*) \neq (s,T) \land (c^*,\tau^*) = (c,\tau')$, there must be a $pk'$ defined on the $P'$ side. If there is a unique record $\langle\pw^*,pk',\star\rangle$, and $\pw^* = \pw$, set $SK := \Hkey(K')$. (In some sense this is the most crucial new game, as it covers the scenario in \Cref{fig:attack-no-pw} --- which is why we need to make all these changes to begin with. It also corresponds to step 2(c)(ii) in \Cref{fig:oeke-sim-hash-pk}. We will deal with step 2(c)(i) later.)

We first argue $\pw = \pw'$ with negligible probability.

Since $\pw \neq \pw' \land (s^*,T^*) \neq (s,T) \land (c^*,\tau^*) = (c,\tau')$, Game 16 finds $\adv$ or $P'$'s $\Htag$ query resulting in $\tau^*$ (see the description of Game 13). $P'$ makes a singe query to $\Htag$, on input $(pk',K')$; and the query result is $\tau' = \tau^*$. Therefore, Game 16 finds this query, and does the following (again see the description of Game 13):
\begin{enumerate}
  \item Find the (unique) record $\langle\pw^*,pk',sk'\rangle$.
  \item Check if $\pw^* = \pw$, which is indeed the case.
  \item Compute $K := \Decap(sk',c^*)$ and check if $K = K'$. We argue that the check must pass. Steps 1 and 2 above mean that there is a (unique) $\langle\pw,pk',sk'\rangle$ record. But then
      \begin{gather*}
      (c,K') \gets \Encap(pk')\text{ on the }P'\text{ side}, \\
      c^* = c, \\
      K = \Decap(sk',c^*)\text{ on the }P\text{ side},
      \end{gather*}
      and $sk'$ is the KEM secret key corresponding to $pk'$ (again, because there is a $\langle\pw,pk',sk'\rangle$ record). So $K = K'$ by the correctness property of KEM.
  \item Set $SK := \Hkey(K) (= \Hkey(K'))$.
\end{enumerate}
This entire process in Game 16 eventually results in setting $SK := \Hkey(K')$, which exactly matches the new game. Thus, there is no difference. We have
\[
\Dist_\env^{16,\text{new}1} = 0.
\]

\noindent\emph{New game 2:} In the case where $\pw \neq \pw' \land (s^*,T^*) \neq (s,T) \land c^* = c$, we find $\adv$ \emph{or $P'$}'s $\Htag(pk^*,K^*)$ query resulting in $\tau^*$ (except that in new game 1's condition, new game 1 already shortcuts it to $SK := \Hkey(K')$ without trying to find any $\Htag$ query). In this game, we do not check $P'$'s $\Htag$ query anymore, and only check $\adv$'s $\Htag$ query.

There are the following cases:
\begin{itemize}
  \item If $\tau^* \neq \tau'$, then of course $P'$'s $\Htag$ query does not result in $\tau^*$ (it results in $\tau'$), so we can safely drop $P'$'s $\Htag$ query.
  \item If $(\pw')^* \neq \pw' \land \tau^* = \tau'$, then $P'$'s $\Htag(pk',K')$ query results in $\tau^*$, and we need to argue that this query can be dropped from the check.
      \begin{itemize}
        \item If $\adv$ queries $\Htag(pk',K')$, then new game 2 still finds this query, so new game 1 and new game 2 are identical.
        \item If $\adv$ does not query $\Htag(pk',K')$, then new game 2 sets $SK := \bot$. We argue that new game 1 also sets $SK := \bot$ except with negligible probability. New game 1 finds $P'$'s $\Htag(pk',K')$ query, then finds the $\langle\pw^*,pk',sk'\rangle$ record, and sets $SK := \bot$ unless $\pw^* = \pw \land \Decap(sk',c^*) = K'$. We claim that $\pw^* = \pw$ with negligible probability; in other words, suppose there is a record $\langle\pw,pk,\star\rangle$, then $pk = pk'$ with negligible probability. \\
            Note that $(pk,\star) \gets \Gen$ and later $H_1(\pw,r)$ is programmed to $T/pk$; for $pk'$, we have
            \[
            pk' = \frac{T^*}{R'} = \frac{T^*}{H_1(\pw',r')}.
            \]
            $R'$ is independent of $pk$, since $\pw \neq \pw'$ by the condition of this game and thus $R' = H_1(\pw',r')$ is independent of $H_1(\pw,r)$. Furthermore, $R'$ is also independent of $T^*$, since $T^*$ is determined \emph{before} the $H_1(\pw',r')$ query:
            \begin{itemize}
              \item If the $H_1(\pw',r')$ query is made by $P'$, then $P'$ makes this query only after receiving $T^*$, so $T^*$ is determined before the $H_1(\pw',r')$ query.
              \item If the $H_1(\pw',r')$ query is made by $\adv$, then $\adv$'s $H_2(\pw',T^*)$ query must come before $\adv$'s $H_1(\pw',r')$ query (otherwise $\pw'$ falls into type (1) in Game 15, so $(\pw')^* = \pw'$ which violates the condition of this game), and $T^*$ must be determined before $\adv$'s $H_2(\pw',T^*)$ query, so $T^*$ is determined before the $H_1(\pw',r')$ query.
            \end{itemize}
            Since $R'$ is independent of both $pk$ and $R'$, $pk' = T^*/R'$ is independent of $pk$ no matter what the distribution of $T^*$ is. Therefore, $\Pr[pk = pk']$ is upper-bounded by $1/|\group|$ for a single $H_1$ query and $(q_1+1)/|\group|$ overall.
      \end{itemize}
  \item If $(\pw')^* = \pw' \land \tau^* = \tau'$, but there is no or more than one $\langle\pw^*,pk',\star\rangle$ record, then both new game 1 and new game 2 outright sets $SK := \bot$, regardless of $\Htag$ queries by $\adv$ or $P'$. Thus, the two games are identical.
  \item If $(\pw')^* = \pw' \land \tau^* = \tau'$, there is a unique $\langle\pw^*,pk',\star\rangle$ record, but $\pw^* \neq \pw$, then again both new game 1 and new game 2 outright sets $SK := \bot$, regardless of $\Htag$ queries by $\adv$ or $P'$. Thus, the two games are identical.
\end{itemize}

The remaining case is that $(\pw')^* = \pw' \land \tau^* = \tau'$, there is a unique $\langle\pw^*,pk',\star\rangle$ record, and $\pw^* = \pw$. This is exactly the condition in new game 1, and new game 1 outright sets $SK := \Hkey(K')$ without trying to find any $\Htag$ query (by $\adv$ or $P'$), so the game change does not affect this case. Overall, we conclude that
\[
\Dist_\env^{\text{new}1,\text{new}2} \leq \frac{q_1+1}{|\group|}.
\]}

\noindent\emph{Game 17:} Note that after new game 2, we never try to find $P'$'s $\Htag$ query anymore: in some cases we try to find $\adv$'s $\Htag$ query, and in other cases we outright set $SK$ to be a certain value or $\bot$. Therefore, the reduction still goes through (it can simulate the $P$ side honestly).

{\color{blue}
\medskip\noindent\emph{New game 3:} In the case where $(\pw')^* = \pw' \land (s^*,T^*) \neq (s,T) \land (c^*,\tau^*) = (c,\allowbreak\tau')$, there is a unique record $\langle\pw^*,pk',\star\rangle$, and $\pw^* \neq \pw$, set $SK := \bot$. (This corresponds to step 2(c)(i) in \Cref{fig:oeke-sim-hash-pk}.)

In this case Game 17 first tries to find $\adv$'s (unique) $\Htag(pk^*,K^*)$ query resulting in $\tau^*$. Since $\tau^* = \tau' = \Htag(pk',K')$, we have $pk^* = pk'$. Then Game 17 tries to find the (unique) $\langle\pw^*,pk',\star\rangle$ record, and checks if $\pw^* = \pw$ (see Game 13). Since $\pw^* \neq \pw$, the check never passes, so both Game 17 and new game 3 set $SK := \bot$. We have
\[
\Dist_\env^{17,\text{new}3} = 0.
\]

\paragraph{Summary.}
We have made the following essential changes to the games:
\begin{itemize}
  \item In the case where $\pw \neq \pw' \land (s^*,T^*) \neq (s,T) \land c^* = c$, Game 2 tries to find $P'$'s $\Htag$ query resulting in $\tau^*$ (in addition to $\adv$'s query).
  \item In the case where $(\pw')^* = \pw' \land (s^*,T^*) \neq (s,T) \land (c^*,\tau^*) = (c,\tau')$, there is a unique record $\langle\pw^*,pk',\star\rangle$, and $\pw^* = \pw$, new game 1 outright sets $SK := \Hkey(K')$ (without trying to find any $\Htag$ query by $\adv$ or $P'$).
  \item In the case where $\pw \neq \pw' \land (s^*,T^*) \neq (s,T) \land c^* = c$, new game 2 drops the check on $P'$'s $\Htag$ query introduced in Game 2 (except in new game 1's condition, where all checks on $\Htag$ queries have already been dropped).
  \item In the case where $(\pw')^* = \pw' \land (s^*,T^*) \neq (s,T) \land (c^*,\tau^*) = (c,\tau')$, there is a unique record $\langle\pw^*,pk',\star\rangle$, and $\pw^* \neq \pw$, new game 3 outright sets $SK := \bot$.
\end{itemize}
